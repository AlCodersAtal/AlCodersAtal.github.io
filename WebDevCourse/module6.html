CSS Grid Basics

CSS Grid is a two-dimensional layout system — perfect for building complex page layouts.

Enabling Grid

CSS Grid
.container { display: grid; grid-template-columns: 200px 200px 200px; /* 3 fixed columns */ grid-template-columns: 1fr 1fr 1fr; /* 3 equal columns */ grid-template-columns: repeat(3, 1fr); /* same, shorthand */ grid-template-columns: 2fr 1fr; /* 2:1 ratio */ gap: 16px; }

Placing Items on the Grid

Grid lets you place items precisely — spanning multiple columns and rows to create complex layouts.

Spanning Columns and Rows

Grid Placement
.item-a { grid-column: 1 / 3; /* span from col 1 to 3 */ grid-column: span 2; /* span 2 columns */ grid-row: 1 / 3; /* span from row 1 to 3 */ } /* Named areas */ .container { grid-template-areas: "header header" "sidebar main" "footer footer"; } header { grid-area: header; } .sidebar { grid-area: sidebar; } main { grid-area: main; } footer { grid-area: footer; }

Media Queries

Media queries let you change your layout based on the screen size — making your site work on any device.

@media Syntax

Media Queries
/* Applies when screen is 768px or smaller */ @media (max-width: 768px) { .container { flex-direction: column; } nav ul { display: none; } } /* Applies when screen is 1024px or wider */ @media (min-width: 1024px) { .grid { grid-template-columns: repeat(4, 1fr); } }
ℹ️ Common breakpoints
  • 480px — small phones
  • 768px — tablets
  • 1024px — laptops
  • 1280px — desktops

Responsive Units

Pixels are fixed. Responsive units scale with the screen — giving you fluid, flexible layouts.

Relative Units

Responsive Units
/* % — relative to parent */ .box { width: 50%; } /* vw / vh — % of viewport width/height */ .hero { width: 100vw; height: 100vh; } /* rem — relative to root font size (usually 16px) */ h1 { font-size: 2rem; } /* 32px */ /* em — relative to parent's font size */ p { font-size: 1em; } /* clamp() — min, preferred, max */ h1 { font-size: clamp(1.5rem, 4vw, 3rem); }
💡 Use rem for fonts, % for widths
This combination gives you consistent scaling across all screen sizes.

Mobile-First Design

Mobile-first means designing for small screens first, then adding styles for larger screens. It's the professional standard.

Mobile-First Approach

Write your base styles for mobile (small screen), then use min-width media queries to enhance for larger screens:

Mobile-First
/* Base styles — mobile (default) */ .grid { display: grid; grid-template-columns: 1fr; /* 1 column */ gap: 16px; } /* Tablet and up */ @media (min-width: 600px) { .grid { grid-template-columns: repeat(2, 1fr); } } /* Desktop and up */ @media (min-width: 1024px) { .grid { grid-template-columns: repeat(3, 1fr); } }
💡 Always add this to your <head>
<meta name="viewport" content="width=device-width, initial-scale=1.0"> — without this, your responsive design won't work on real phones!
