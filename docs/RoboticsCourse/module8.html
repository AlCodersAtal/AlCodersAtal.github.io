Decision Making in Code

A robot's "intelligence" is just well-structured decisions. Map behaviour to code before writing a single line.

Draw a Decision Tree First

Decision Tree (Plain English)
Is distance < 15cm? YES → Stop → Reverse 500ms → Scan left & right Is left distance > right distance? YES → Spin Left 450ms → Drive Forward NO → Spin Right 450ms → Drive Forward NO → Drive Forward at full speed

This tree translates almost directly into the state machine code you'll write in the final project.

Non-Blocking Timing with millis()

delay() freezes the entire Arduino — your sensor goes unread. Use millis() instead:

Arduino C++
unsigned long timer = 0; bool turning = false; void loop() { if (turning) { if (millis() - timer >= 450) { turning=false; stopMotors(); } } else { if (getDistance() < 15) { spinRight(180); timer=millis(); turning=true; } else { driveForward(200); } } }

State Machines

A state machine organises behaviour into clear "states" — making complex programs easy to read and debug.

At any time the robot is in exactly one state. Events (sensor readings, timers) cause transitions to other states.

Arduino C++ — State Machine Pattern
#define ST_FORWARD 0 #define ST_REVERSE 1 #define ST_TURN 2 int state = ST_FORWARD; unsigned long timer = 0; int dir = 1; // 1=right, -1=left void loop() { switch(state) { case ST_FORWARD: driveForward(200); if (getDistance() < 15) { stopMotors(); state=ST_REVERSE; timer=millis(); } break; case ST_REVERSE: driveBackward(150); if (millis()-timer > 500) { stopMotors(); dir=scanBestDir(); state=ST_TURN; timer=millis(); } break; case ST_TURN: dir==1 ? spinRight(180) : spinLeft(180); if (millis()-timer > 450) { stopMotors(); state=ST_FORWARD; } break; } }

Debugging Techniques

Every robot builder spends time debugging. These techniques save hours of frustration.

Tool #1: Serial Monitor

Arduino C++
void loop() { float d = getDistance(); Serial.print("State: "); Serial.print(state); Serial.print(" Distance: "); Serial.println(d); // Now you can see EXACTLY what the robot sees }

Common Problems & Fixes

SymptomLikely CauseFix
Robot doesn't moveL298N not powered / wrong wiringCheck battery, common GND, IN pins
Spins in circlesOne motor wired backwardsSwap OUT1/OUT2 for that motor
Sensor reads 0 alwaysTRIG/ECHO swapped or no timeoutCheck wiring, add 30000µs timeout
Arduino resets randomlyPower brownout from motorsAdd 100µF cap across motor power lines
Servo jittersNoise on 5V railAdd 10µF cap between servo VCC and GND
💡 Debug One Subsystem at a Time
Test sensor alone first (print readings). Then test motors alone. Then combine. Never debug everything at once.
