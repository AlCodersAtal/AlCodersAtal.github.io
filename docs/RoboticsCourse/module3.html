Meet the Arduino Uno

The Arduino Uno is the most popular microcontroller board in the world — and the brain of your robot.

Key Parts of the Arduino Uno

PartWhat It Is
ATmega328P chipThe microcontroller — where your code runs at 16 MHz
Digital Pins 0–13Can be INPUT or OUTPUT — read sensors, drive LEDs & motors
Analog Pins A0–A5Read 0–1023 values from sensors (0–5V range)
PWM Pins ~3,5,6,9,10,11Simulate analog output — motor speed & servo control
5V & 3.3V pinsPower output rails for connected components
GND pinsGround — the reference for all voltages
USB PortUpload code from your computer, also powers the board
VIN / Barrel JackPower from a battery (7–12V recommended)
Reset ButtonRestarts your sketch from the beginning
⚠️ Current Limits
Each digital pin can safely provide 40mA max. All pins combined should not exceed 200mA. This is why motors must always use a separate driver board.

Arduino IDE Setup

The Arduino IDE is your coding environment — where you write, verify, and upload sketches to the board.

Installation Steps

  1. Go to arduino.cc/en/software and download Arduino IDE 2.x for your operating system.
  2. Run the installer with default settings. Accept any USB driver prompts on Windows.
  3. Connect your Arduino Uno via USB cable.
  4. In the IDE: Tools → Board → Arduino AVR Boards → Arduino Uno.
  5. Go to Tools → Port and select your COM port (Windows: COM3/COM4 · Mac: /dev/cu.usbmodem…).
  6. Open File → Examples → 01.Basics → Blink and click the Upload arrow (→).

If the onboard LED next to pin 13 starts blinking — your setup is working!

💡 Port Not Showing?
Many cheap USB cables are charge-only and carry no data. Try a different cable. If still stuck, install the CH340 driver (needed for some Arduino clone boards).

IDE Anatomy

  • Verify (✓) — Compiles your code and checks for errors. Does not upload.
  • Upload (→) — Compiles and sends code to the board.
  • Serial Monitor — Shows output from Serial.print(). Essential for debugging.
  • Serial Plotter — Graphs numeric sensor data in real time.
Module 03 · Lesson 3

Digital Pins & Blink

Digital pins are binary — HIGH (5V) or LOW (0V). Controlling them is how you switch things on and off.

The Blink Sketch — Explained Line by Line

Arduino C++
#define LED_PIN 13 // Onboard LED is on pin 13 void setup() { // Runs ONCE when the board powers on pinMode(LED_PIN, OUTPUT); } void loop() { // Runs FOREVER in a cycle digitalWrite(LED_PIN, HIGH); // Turn LED ON (5V) delay(1000); // Wait 1 second digitalWrite(LED_PIN, LOW); // Turn LED OFF (0V) delay(1000); }
  • setup() — Runs once on power-on. Configure pins and start Serial here.
  • loop() — Runs forever. Your main robot logic lives here.
  • pinMode() — Tells a pin whether it's an INPUT or OUTPUT.
  • digitalWrite() — Sets a pin HIGH (5V) or LOW (0V).
  • delay(ms) — Pauses execution for the given number of milliseconds.
💡 Challenge: SOS Blink
Modify the sketch to blink SOS in Morse code: 3 short (200ms on) · 3 long (600ms on) · 3 short (200ms on). Add 400ms gaps between groups.

Analog Pins & Serial Monitor

Analog pins read continuous values — like light levels, temperature, or potentiometer position.

analogRead()

The analog pins (A0–A5) convert voltage (0–5V) into a number from 0 to 1023 using a built-in 10-bit ADC.

Arduino C++
void setup() { Serial.begin(9600); // Start Serial communication at 9600 baud } void loop() { int raw = analogRead(A0); // Read pin A0 → 0 to 1023 float voltage = raw * (5.0 / 1023.0); // Convert to volts Serial.print("Raw: "); Serial.print(raw); Serial.print(" Voltage: "); Serial.println(voltage); delay(500); }

Open the Serial Monitor (Ctrl+Shift+M) and set baud rate to 9600. You'll see live readings every 500ms.
