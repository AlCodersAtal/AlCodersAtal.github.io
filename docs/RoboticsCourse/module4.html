Variables & Data Types

Variables are named containers for data. Choosing the right type keeps your code efficient and bug-free.

TypeSizeRangeUse In Robotics
int2 bytes−32,768 to 32,767Pin numbers, counts, PWM values
long4 bytes±2 billionmillis() timestamps, pulse durations
float4 bytesDecimal numbersDistance in cm, voltages
bool1 bytetrue / falseObstacle detected? Robot turning?
const int2 bytesFixed valuePin numbers, thresholds
Arduino C++
// Variable declarations const int TRIG_PIN = 11; // const: value never changes const int ECHO_PIN = 12; const float STOP_DIST = 15.0; // cm — obstacle threshold float distance = 0.0; // updated each loop bool obstacle = false; // flag unsigned long timer = 0; // for millis() timing
💡 Use const
Declare pin numbers and thresholds as const. It makes your code self-documenting and the compiler can optimise it.
Module 04 · Lesson 2

Conditionals & Loops

Conditionals let your robot make decisions. Loops repeat actions. Together they power all robot behaviour.

Arduino C++ — Conditionals
// if / else if / else if (distance < 10) { stopMotors(); reverseMotors(); } else if (distance < 20) { slowDown(); } else { driveForward(); }
Arduino C++ — Loops
// for loop — blink LED 5 times for (int i = 0; i < 5; i++) { digitalWrite(LED_PIN, HIGH); delay(200); digitalWrite(LED_PIN, LOW); delay(200); } // while loop — keep reversing until clear while (distance < 30) { reverseMotors(); distance = getDistance(); }
⚠️ Avoid Blocking Loops
while inside loop() blocks everything else. In advanced programs, prefer millis() for non-blocking timing.
Module 04 · Lesson 3

Functions

Functions let you name and reuse code — making programs readable, modular, and easy to debug.

Arduino C++
// Function that RETURNS a value float getDistance() { digitalWrite(TRIG_PIN, LOW); delayMicroseconds(2); digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10); digitalWrite(TRIG_PIN, LOW); long dur = pulseIn(ECHO_PIN, HIGH, 30000); return dur ? dur * 0.034 / 2.0 : 999; } // Void function — does something, returns nothing void driveForward(int speed) { analogWrite(ENA, speed); analogWrite(ENB, speed); digitalWrite(IN1, HIGH); digitalWrite(IN2, LOW); digitalWrite(IN3, HIGH); digitalWrite(IN4, LOW); } // Using them in loop() void loop() { float dist = getDistance(); if (dist > 20) driveForward(180); else stopMotors(); }
💡 Structure Tip
Build your robot program as a library of small, clear functions: driveForward(), spinLeft(), stopMotors(), getDistance(). Your final sketch will read almost like English.
Module 04 · Lesson 4

Libraries

Libraries are pre-written code packages that save you from reinventing the wheel.

Installing a Library

  1. In the IDE: Sketch → Include Library → Manage Libraries…
  2. Search for the library name (e.g. NewPing).
  3. Click Install. The IDE downloads and sets it up automatically.
  4. Add to top of sketch: #include <LibraryName.h>
LibraryPurpose
Servo.hBuilt-in — control servo motors by angle
NewPingCleaner ultrasonic readings with error filtering
Wire.hBuilt-in — I2C for gyros, displays
IRremoteReceive signals from IR remote controls
Arduino C++ — Servo Library
#include <Servo.h> // Built-in, no install needed Servo scanServo; void setup() { scanServo.attach(6); // Signal wire to pin 6 scanServo.write(90); // Centre position }
