Project Overview & Planning

Everything you've learned — electronics, Arduino, sensors, motors, state machines — comes together into one real autonomous robot.

🤖 What You're Building

A servo-scanning obstacle-avoiding robot that navigates freely without human input.

  1. Drives forward autonomously at full speed
  2. Detects obstacles with HC-SR04 mounted on SG90 servo
  3. Stops and reverses for 500ms when obstacle < 15cm
  4. Pans servo left and right to measure open space
  5. Turns toward the clearer side
  6. Resumes forward driving — forever

Hardware Assembly

Step-by-step assembly. Work slowly — a neat build is a debuggable build.

  1. Chassis — Assemble the acrylic frame. Mount TT motors to rear brackets. Attach wheels. Install caster at the front.
  2. Mount L298N — Attach to the top plate. Connect motor wires: Left motor → OUT1/OUT2, Right → OUT3/OUT4.
  3. Mount Arduino — Secure to top plate. Connect a common GND wire between Arduino GND and L298N GND.
  4. Servo mount — Attach SG90 to the front of the chassis using a servo bracket. Mount HC-SR04 to the servo horn facing forward.
  5. Battery — Secure the battery holder underneath or at the rear for low centre of gravity.
  6. Wire everything — Follow the Master Wiring Table from Module 7 exactly. Use short jumper wires. Secure loose cables with a zip tie.
  7. Power test (no code yet) — Connect battery. L298N and Arduino power LEDs should light. If anything smells hot, disconnect immediately and check for short circuits.
⚠️ Do a Motor Spin Test First
Before full assembly, upload a simple motor test sketch to verify each motor spins in the correct direction. Much easier to swap wires before everything is bolted down.

Arduino C++ — Full Robot Sketch
// ═══════════════════════════════════════════════════ // AlCoders ATL — Obstacle-Avoiding Robot // Module 9 Final Project // ═══════════════════════════════════════════════════ #include <Servo.h> // ── Pin Definitions ────────────────────────────── #define IN1 2 #define IN2 3 #define IN3 4 #define IN4 5 #define ENA 9 #define ENB 10 #define TRIG 11 #define ECHO 12 #define SERVO_PIN 6 // ── Tuning — adjust these to improve behaviour ─── const int DRIVE_SPD = 200; // 0–255 forward speed const int TURN_SPD = 180; // spin speed const int REV_SPD = 150; // reverse speed const float STOP_DIST = 15.0; // cm — stop threshold const long REV_TIME = 500; // ms to reverse const long TURN_TIME = 450; // ms to turn // ── States ─────────────────────────────────────── #define ST_FORWARD 0 #define ST_REVERSE 1 #define ST_TURN 2 int state = ST_FORWARD; unsigned long timer = 0; int dir = 1; Servo servo; // ── Setup ──────────────────────────────────────── void setup() { pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT); pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT); pinMode(ENA,OUTPUT); pinMode(ENB,OUTPUT); pinMode(TRIG,OUTPUT); pinMode(ECHO,INPUT); servo.attach(SERVO_PIN); servo.write(90); delay(500); Serial.begin(9600); Serial.println("AlCoders Robot — Ready!"); } // ── Distance Sensor ────────────────────────────── float getDist() { digitalWrite(TRIG,LOW); delayMicroseconds(2); digitalWrite(TRIG,HIGH); delayMicroseconds(10); digitalWrite(TRIG,LOW); long d = pulseIn(ECHO,HIGH,30000); return d ? d*0.034/2.0 : 200; } // ── Motor Functions ────────────────────────────── void fwd(int s){analogWrite(ENA,s);analogWrite(ENB,s);digitalWrite(IN1,1);digitalWrite(IN2,0);digitalWrite(IN3,1);digitalWrite(IN4,0);} void rev(int s){analogWrite(ENA,s);analogWrite(ENB,s);digitalWrite(IN1,0);digitalWrite(IN2,1);digitalWrite(IN3,0);digitalWrite(IN4,1);} void sRight(int s){analogWrite(ENA,s);analogWrite(ENB,s);digitalWrite(IN1,1);digitalWrite(IN2,0);digitalWrite(IN3,0);digitalWrite(IN4,1);} void sLeft(int s){analogWrite(ENA,s);analogWrite(ENB,s);digitalWrite(IN1,0);digitalWrite(IN2,1);digitalWrite(IN3,1);digitalWrite(IN4,0);} void stop(){analogWrite(ENA,0);analogWrite(ENB,0);} // ── Scan: pan servo, return best direction ──────── int scan() { servo.write(10); delay(400); float L = getDist(); servo.write(170); delay(400); float R = getDist(); servo.write(90); delay(300); Serial.print("L:"); Serial.print(L); Serial.print(" R:"); Serial.println(R); return (L >= R) ? -1 : 1; // -1=left, 1=right } // ── Main Loop ──────────────────────────────────── void loop() { switch(state) { case ST_FORWARD: { fwd(DRIVE_SPD); float d = getDist(); Serial.print("Fwd:"); Serial.println(d); if (d < STOP_DIST) { stop(); state=ST_REVERSE; timer=millis(); } break; } case ST_REVERSE: rev(REV_SPD); if (millis()-timer >= REV_TIME) { stop(); delay(100); dir=scan(); state=ST_TURN; timer=millis(); } break; case ST_TURN: dir==1 ? sRight(TURN_SPD) : sLeft(TURN_SPD); if (millis()-timer >= TURN_TIME) { stop(); state=ST_FORWARD; } break; } }

Testing & Improvements

Your robot is alive! Now tune it, stress-test it, and push it further.

Testing Sequence

  1. Upload & open Serial Monitor — confirm distance readings are sensible before letting it drive.
  2. Bench test (held up) — verify all 4 motor functions: forward, backward, spin left, spin right.
  3. Hand obstacle test — place your hand 8 cm in front. Robot should stop, reverse, scan, and turn.
  4. Open floor test — let it run freely for 2 minutes. Watch for unexpected circles or stalls.
  5. Obstacle course — set up boxes in a maze. Robot should navigate autonomously.

Tuning Constants

ConstantEffect of Increasing
STOP_DISTRobot stops further from obstacles (safer, slower)
DRIVE_SPDRobot drives faster (harder to stop in time)
REV_TIMERobot backs up further before scanning
TURN_TIMERobot turns further before resuming

Ideas for Next Steps

  • Add a buzzer that beeps when an obstacle is detected.
  • Add a Bluetooth HC-05 module for remote override from your phone.
  • Add IR line sensors below the chassis for line-following mode.
  • Replace the timed turn with encoder feedback for precise angles.
  • Add an OLED display showing current state and distance.
