Interrupts, Timers & Power

Learn to respond to events instantly, use hardware timers for precise timing, and put your Arduino to sleep to save power.

⚡
Lesson 1 of 5

What are Interrupts?

Normally Arduino runs code line by line in a loop. Interrupts break this pattern — they immediately pause whatever the processor is doing and run a special function called an ISR (Interrupt Service Routine) the instant an event occurs.

Why Use Interrupts?

Imagine you need to count button presses while also running other code. If you check the button inside loop(), you might miss fast presses. An interrupt fires the moment the button is pressed — no polling needed, no misses.

MethodPolling (loop)Interrupt
Response timeDepends on loop speedInstant (microseconds)
CPU usageWastes cycles checkingEfficient — only runs on event
Best forSlow, non-critical eventsFast, critical events
ExampleRead temperature every secondCount encoder pulses

Interrupt Sources on Arduino Uno

Arduino Mega has 6 external interrupt pins (2, 3, 18, 19, 20, 21). Arduino Leonardo uses different pin numbers — always check your board's documentation.

Lesson 2 of 5

attachInterrupt()

The attachInterrupt() function registers an ISR to run when a specific event occurs on an interrupt pin. The ISR must be fast — no delay(), no Serial prints, no heavy computation.

Arduino C++ — Button Counter with Interrupt
volatile int count = 0;  // volatile! ISR modifies this
volatile bool pressed = false;

void buttonISR() {
  count++;          // Increment counter
  pressed = true;  // Set flag for main loop
}

void setup() {
  Serial.begin(9600);
  pinMode(2, INPUT_PULLUP);

  // attachInterrupt(digitalPinToInterrupt(pin), ISR, mode)
  attachInterrupt(digitalPinToInterrupt(2), buttonISR, FALLING);
  // FALLING = trigger when pin goes HIGH→LOW (button press)
}

void loop() {
  if (pressed) {
    pressed = false;            // Clear flag
    Serial.print("Count: ");
    Serial.println(count);
  }
  // Other code runs here without missing button presses
  delay(100);
}

Interrupt Modes

ModeTriggers When
RISINGPin changes from LOW → HIGH
FALLINGPin changes from HIGH → LOW
CHANGEPin changes in either direction
LOWPin is LOW (continuous, use carefully)
⚠️
ISR Rules — Critical!
  • Always declare shared variables as volatile
  • Never use delay() inside an ISR
  • Never use Serial.print() inside an ISR
  • Keep ISRs as short as possible — set a flag, do work in loop()
  • millis() won't increment inside an ISR

Hardware Timers

The Arduino Uno's ATmega328P has 3 hardware timers (Timer0, Timer1, Timer2). These are independent counters that count clock cycles — they keep time accurately even while your code is doing something else.

The Three Timers

TimerBitsUsed ByPeriod
Timer08-bitdelay(), millis(), micros()~1ms
Timer116-bitServo library, tone()Up to 4s
Timer28-bittone() on some boards~16ms
⚠️
Timer Conflicts The Servo library uses Timer1. If you use Servo and try to generate tones on pins 9/10 (which also use Timer1), there will be conflicts. Plan your pin and library usage carefully.

Using the TimerOne Library

For precise, repeating timer interrupts, use the TimerOne library (install via Library Manager). This fires an ISR at a fixed interval — perfect for PID control, sensor polling, or anything needing exact timing.

Arduino C++ — TimerOne: Blink LED every 500ms precisely
#include <TimerOne.h>

volatile bool ledState = false;

void timerISR() {
  ledState = !ledState;
  digitalWrite(13, ledState);
}

void setup() {
  pinMode(13, OUTPUT);
  Timer1.initialize(500000);   // 500,000 microseconds = 500ms
  Timer1.attachInterrupt(timerISR);
}

void loop() {
  // Main loop free to do other tasks
  // LED blinks precisely every 500ms regardless
}

millis() vs delay() — Non-Blocking Code

This is one of the most important concepts in embedded programming. delay() freezes everything — no sensor reads, no button checks, nothing. millis() lets you time events without blocking anything.

The Problem with delay()

Arduino C++ — WRONG: delay() blocks everything
void loop() {
  // Blink LED every second
  digitalWrite(13, HIGH);
  delay(1000);  // Arduino FROZEN for 1 second!
  digitalWrite(13, LOW);
  delay(1000);  // FROZEN again!
  // During these 2 seconds, you can't read sensors,
  // check buttons, update display, or do ANYTHING ELSE
}

The millis() Solution

Arduino C++ — RIGHT: millis() non-blocking pattern
unsigned long prevLED = 0;
unsigned long prevSensor = 0;
bool ledState = false;

void loop() {
  unsigned long now = millis();

  // Blink LED every 1000ms
  if (now - prevLED >= 1000) {
    prevLED = now;
    ledState = !ledState;
    digitalWrite(13, ledState);
  }

  // Read sensor every 200ms
  if (now - prevSensor >= 200) {
    prevSensor = now;
    int val = analogRead(A0);
    Serial.println(val);
  }

  // Both tasks run SIMULTANEOUSLY — no blocking!
  // Check buttons, update display, etc. here too
}
💡
millis() Overflow millis() rolls over to 0 after ~49 days. The pattern now - prevTime >= interval handles this correctly because unsigned subtraction wraps around properly. Never use millis() >= prevTime + interval — that breaks at overflow!

Sleep Modes & Power Saving

Battery-powered Arduino projects live or die on power consumption. The ATmega328P normally draws ~15-20mA, but using sleep modes you can drop this to under 1µA — extending battery life from days to years.

Arduino Sleep Modes

ModeCurrent DrawWake SourcesUse Case
Idle~6mAAny interruptLight sleep, timers still run
ADC Noise Reduction~1.1mAADC complete, INT0Precise ADC reads
Power-save~1.1mATimer2, TWI, INTTimer2 still runs
Power-down~0.1µAExternal interrupt, WDTDeepest sleep, lowest power
Standby~0.8µAINT, crystal oscillator stays onFast wake needed
Arduino C++ — Deep Sleep, wake on button press
#include <avr/sleep.h>
#include <avr/power.h>

void wakeISR() {
  // Woke up! Just return, setup will continue
}

void goToSleep() {
  sleep_enable();
  attachInterrupt(digitalPinToInterrupt(2), wakeISR, LOW);
  set_sleep_mode(SLEEP_MODE_PWR_DOWN);   // Deepest sleep
  sleep_cpu();                           // ZZZzzz...
  // --- CODE RESUMES HERE AFTER WAKE ---
  sleep_disable();
  detachInterrupt(digitalPinToInterrupt(2));
}

void setup() {
  Serial.begin(9600);
  pinMode(2, INPUT_PULLUP);
  pinMode(13, OUTPUT);
}

void loop() {
  Serial.println("Awake! Doing work...");
  digitalWrite(13, HIGH);
  delay(1000);
  digitalWrite(13, LOW);

  Serial.println("Going to sleep. Press button to wake.");
  Serial.flush();   // Finish sending before sleep!
  goToSleep();      // Sleep until button press
}
💡
More Power Savings Disable unused peripherals using the power reduction register: power_adc_disable(), power_spi_disable(), power_twi_disable(). Together these can reduce active current from ~15mA to under 5mA even when awake.
