🔌 Lesson 1: Digital Pins Explained

Arduino's digital pins are the most fundamental I/O mechanism. Each pin can be in one of two states: HIGH (5V) or LOW (0V). There are 14 digital pins (0–13) on the Arduino Uno.

Digital vs Analog Thinking

Digital is binary — it's either ON or OFF. Light switches, buttons, and relay signals are digital. There's no "in between." This makes digital I/O simple to code but limited in expressiveness compared to analog (which comes in Module 4).

Pin Modes

ModeUsageDescription
OUTPUTDriving LEDs, relays, buzzersPin outputs 5V (HIGH) or 0V (LOW)
INPUTReading sensors, buttonsPin reads external voltage — floating if not connected!
INPUT_PULLUPButtons (recommended)Internal 20kΩ pull-up resistor activated — pin reads HIGH when open, LOW when button pressed
⚠️
Floating PinsAn INPUT pin with nothing connected to it will "float" — it randomly reads HIGH or LOW. Always connect a pull-up or pull-down resistor, or use INPUT_PULLUP mode.
Arduino C++
void setup() {
  pinMode(13, OUTPUT);       // pin 13 as output
  pinMode(2, INPUT_PULLUP);   // pin 2 as input with pull-up
  Serial.begin(9600);
}

void loop() {
  int state = digitalRead(2);  // Read pin 2
  Serial.println(state);         // Print to Serial Monitor
  delay(100);
}

⬆️ Lesson 2: digitalWrite & digitalRead

The two most essential digital functions — mastering these lets you control outputs and read inputs.

digitalWrite — Controlling Outputs

Arduino C++
// digitalWrite(pin, value)
// value = HIGH or LOW (or 1 or 0)

digitalWrite(13, HIGH);   // Set pin 13 to 5V — LED ON
digitalWrite(13, LOW);    // Set pin 13 to 0V — LED OFF

// You can also use 1 and 0:
digitalWrite(13, 1);     // Same as HIGH
digitalWrite(13, 0);     // Same as LOW

digitalRead — Reading Inputs

Arduino C++
// digitalRead(pin) returns HIGH or LOW
int buttonState = digitalRead(2);

if (buttonState == HIGH) {
  Serial.println("Button is NOT pressed");
} else {
  Serial.println("Button IS pressed!");
  digitalWrite(13, HIGH);  // Turn on LED
}
📝
Why is button press LOW?With INPUT_PULLUP, the pin is pulled to HIGH (5V) by default. When you press the button and connect the pin to GND, it becomes LOW. So pressed = LOW, released = HIGH. This is counterintuitive at first but becomes second nature!

🔘 Lesson 3: Push Buttons & Input

Push buttons are momentary switches — they only connect while you hold them. Here's how to wire and read them correctly.

Wiring a Button with INPUT_PULLUP

Button PinArduino Pin
Pin 1 (any leg)Digital Pin 2
Pin 2 (other leg)GND

With INPUT_PULLUP — no external resistor needed! The Arduino's internal resistor handles it.

Arduino C++
#define BUTTON_PIN 2
#define LED_PIN    13

void setup() {
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
}

void loop() {
  // With pullup: button pressed = LOW
  if (digitalRead(BUTTON_PIN) == LOW) {
    digitalWrite(LED_PIN, HIGH);  // LED on while pressing
  } else {
    digitalWrite(LED_PIN, LOW);   // LED off when released
  }
}

🛡️ Lesson 4: Debouncing Buttons

When you press a physical button, the metal contacts don't make a clean single connection. They "bounce" — rapidly connecting and disconnecting dozens of times in a few milliseconds. This causes false multiple presses. Debouncing eliminates this.

The Problem

Without debouncing, pressing a button once might be detected as 5–20 presses. For a counter or toggle, this causes chaos.

Software Debounce Solution

Arduino C++
#define BUTTON_PIN 2
#define LED_PIN    13
#define DEBOUNCE_MS 50  // 50ms debounce window

int ledState = LOW;
int lastButtonState = HIGH;
unsigned long lastDebounceTime = 0;

void setup() {
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP);
}

void loop() {
  int reading = digitalRead(BUTTON_PIN);

  if (reading != lastButtonState) {
    lastDebounceTime = millis();  // Reset timer on change
  }

  if ((millis() - lastDebounceTime) > DEBOUNCE_MS) {
    // State is stable! Act on it.
    if (reading == LOW) {  // Button pressed
      ledState = !ledState;  // Toggle LED
      digitalWrite(LED_PIN, ledState);
    }
  }
  lastButtonState = reading;
  delay(10);
}
💡
Hardware DebounceYou can also add a 100nF capacitor across the button terminals. This smooths the bounce electrically, no code needed. For production designs, hardware debounce is preferred.

🚦 Lesson 5: Project – Traffic Light

🚦 Traffic Light Simulator

Build a traffic light using 3 LEDs (Red, Yellow, Green) that cycles through the traffic light sequence automatically.

Arduino UnoRed LEDYellow LED Green LED3× 220Ω ResistorsBreadboardWires

Wiring

ComponentArduino Pin
Red LED (+)Pin 9
Yellow LED (+)Pin 10
Green LED (+)Pin 11
All LEDs (−) via 220ΩGND
Arduino C++
#define RED    9
#define YELLOW 10
#define GREEN  11

void setup() {
  pinMode(RED, OUTPUT);
  pinMode(YELLOW, OUTPUT);
  pinMode(GREEN, OUTPUT);
}

void setLight(int red, int yellow, int green) {
  digitalWrite(RED, red);
  digitalWrite(YELLOW, yellow);
  digitalWrite(GREEN, green);
}

void loop() {
  setLight(HIGH, LOW, LOW);  // Red ON — Stop
  delay(3000);               // 3 seconds

  setLight(HIGH, HIGH, LOW); // Red+Yellow — Get Ready
  delay(1000);               // 1 second

  setLight(LOW, LOW, HIGH);  // Green ON — Go
  delay(3000);               // 3 seconds

  setLight(LOW, HIGH, LOW);  // Yellow — Slow Down
  delay(1000);               // 1 second
}
