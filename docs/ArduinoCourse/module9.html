HC-05 Bluetooth Module

The HC-05 is the most popular Bluetooth module for Arduino — affordable, easy to use, and works with Android phones out of the box. It operates on Classic Bluetooth 2.0 and creates a simple serial (UART) wireless link.

HC-05 vs HC-06

FeatureHC-05HC-06
ModeMaster & SlaveSlave only
AT CommandsFull supportLimited
PriceSlightly higherCheaper
Best ForComplex projectsSimple phone control

Pinout & Wiring

💡
Voltage Divider on RX! HC-05 RX pin operates at 3.3V but Arduino TX outputs 5V. Use a voltage divider (1kΩ + 2kΩ) to step it down, or you may damage the module over time.
HC-05 PinArduino PinNote
VCC5VPower
GNDGNDGround
TXPin 10 (RX)Direct connection OK
RXPin 11 (TX)Voltage divider needed!
EN/KEYPin 9 (optional)For AT command mode
Arduino C++
#include <SoftwareSerial.h>

// RX=10 (receives from HC-05 TX), TX=11 (sends to HC-05 RX)
SoftwareSerial btSerial(10, 11);

void setup() {
  Serial.begin(9600);      // USB serial for debugging
  btSerial.begin(9600);   // HC-05 default baud rate
  Serial.println("Bluetooth ready!");
}

void loop() {
  // Forward data from Bluetooth to Serial Monitor
  if (btSerial.available()) {
    Serial.write(btSerial.read());
  }
  // Forward data from Serial Monitor to Bluetooth
  if (Serial.available()) {
    btSerial.write(Serial.read());
  }
}

Lesson 2 of 5

AT Commands Setup

AT commands let you configure the HC-05 — change its name, baud rate, pairing PIN, and operating mode. This is done in a special AT Command Mode which you enter by holding the EN/KEY pin HIGH before powering on.

Entering AT Command Mode

  1. Connect the EN/KEY pin to 3.3V (or use the button on the module)
  2. Power on the Arduino
  3. The LED on HC-05 blinks slowly (every 2 seconds) — you're in AT mode
  4. Set your Serial Monitor baud to 38400, line ending to "Both NL & CR"
⚠️
AT Mode Baud Rate In AT mode, HC-05 communicates at 38400 baud — NOT 9600. Your normal sketch uses 9600. These are different!

Essential AT Commands

CommandResponsePurpose
ATOKTest connection
AT+NAME?+NAME:HC-05Get device name
AT+NAME=MyBotOKSet device name
AT+PSWD?+PSWD:1234Get pairing PIN
AT+PSWD=0000OKSet new PIN
AT+UART?+UART:9600,0,0Get baud rate
AT+UART=115200,0,0OKSet baud to 115200
AT+ROLE?+ROLE:00=Slave, 1=Master
AT+RESETOKReboot module
Arduino C++ — AT Mode Sketch
#include <SoftwareSerial.h>
SoftwareSerial btSerial(10, 11);

void setup() {
  Serial.begin(38400);    // AT mode baud rate
  btSerial.begin(38400);
}

void loop() {
  if (btSerial.available())
    Serial.write(btSerial.read());
  if (Serial.available())
    btSerial.write(Serial.read());
}
// Upload this, enter AT mode, open Serial Monitor at 38400
// Type: AT  → should respond: OK

Phone-to-Arduino Control

The most popular use of HC-05: control your Arduino from your Android phone! Use a free app like Bluetooth Terminal HC-05 or Arduino Bluetooth Controller from the Play Store.

How It Works

Your phone sends single characters (like '1', '0', 'F', 'B') over Bluetooth. Arduino receives them on its SoftwareSerial port and reacts accordingly. This is exactly how remote-controlled robots and smart LED controllers work.

💡
Simple Protocol Use single characters as commands — they're fast, easy to parse, and don't get corrupted. Avoid sending full words like "forward" — use 'F' instead.
Arduino C++ — Bluetooth LED Control
#include <SoftwareSerial.h>
SoftwareSerial btSerial(10, 11);

const int LED_RED   = 5;
const int LED_GREEN = 6;
const int LED_BLUE  = 7;

void setup() {
  btSerial.begin(9600);
  pinMode(LED_RED,   OUTPUT);
  pinMode(LED_GREEN, OUTPUT);
  pinMode(LED_BLUE,  OUTPUT);
}

void loop() {
  if (btSerial.available()) {
    char cmd = btSerial.read();

    switch(cmd) {
      case 'R': digitalWrite(LED_RED,   HIGH); break;
      case 'G': digitalWrite(LED_GREEN, HIGH); break;
      case 'B': digitalWrite(LED_BLUE,  HIGH); break;
      case 'X': // Turn all off
        digitalWrite(LED_RED,   LOW);
        digitalWrite(LED_GREEN, LOW);
        digitalWrite(LED_BLUE,  LOW);
        break;
    }
    btSerial.print("CMD:");
    btSerial.println(cmd);   // Echo back to phone
  }
}

Pairing Your Phone

  1. Power up your Arduino + HC-05 (LED blinks fast in normal mode)
  2. On Android: Settings → Bluetooth → Scan for devices
  3. Tap "HC-05" (or whatever name you set)
  4. Enter PIN: 1234 (default) or 0000
  5. Open your Bluetooth terminal app, select HC-05, connect
  6. Send characters like 'R', 'G', 'B', 'X'

BLE with ESP32

Bluetooth Low Energy (BLE) is the modern standard — it uses far less power, works with iPhones (Classic Bluetooth doesn't!), and enables more sophisticated communication. The ESP32 has built-in BLE, making it perfect for BLE projects.

Classic Bluetooth vs BLE

FeatureClassic BluetoothBLE
PowerHigher (~30mA)Very low (~1-15mA)
iOS Support❌ No✅ Yes
Android Support✅ Yes✅ Yes
Data RateUp to 3 MbpsUp to 2 Mbps
Use CaseAudio, continuous streamSensors, IoT, beacons

BLE Concepts: GATT, Services & Characteristics

BLE uses a system called GATT (Generic Attribute Profile). Think of it like a website: a Service is a web page, and Characteristics are the individual fields on that page (text boxes you can read or write).

ESP32 Arduino — BLE Server (simple LED control)
#include <BLEDevice.h>
#include <BLEServer.h>
#include <BLEUtils.h>
#include <BLE2902.h>

#define SERVICE_UUID        "4fafc201-1fb5-459e-8fcc-c5c9c331914b"
#define CHARACTERISTIC_UUID "beb5483e-36e1-4688-b7f5-ea07361b26a8"
#define LED_PIN 2

class LEDCallbacks : public BLECharacteristicCallbacks {
  void onWrite(BLECharacteristic *pC) {
    String val = pC->getValue().c_str();
    if (val == "ON")  digitalWrite(LED_PIN, HIGH);
    if (val == "OFF") digitalWrite(LED_PIN, LOW);
  }
};

void setup() {
  pinMode(LED_PIN, OUTPUT);
  BLEDevice::init("AlCoders-ESP32");
  BLEServer *pServer = BLEDevice::createServer();
  BLEService *pService = pServer->createService(SERVICE_UUID);
  BLECharacteristic *pChar = pService->createCharacteristic(
    CHARACTERISTIC_UUID,
    BLECharacteristic::PROPERTY_WRITE
  );
  pChar->setCallbacks(new LEDCallbacks());
  pService->start();
  pServer->getAdvertising()->start();
}

void loop() { /* BLE handled in background */ }
💡
Testing BLE Use the free app nRF Connect (iOS & Android) to scan for BLE devices, connect, and write to characteristics — perfect for testing without building a custom app.

Bluetooth Car Project

Let's build a phone-controlled robot car using HC-05 + L298N motor driver. You'll send direction commands ('F', 'B', 'L', 'R', 'S') from your phone and the car will move accordingly.

ComponentQty

Jumper wiresSeveral

Motor Driver Pin Mapping

L298N PinArduino PinFunction
IN12Motor A direction 1
IN23Motor A direction 2
IN34Motor B direction 1
IN45Motor B direction 2
ENA6 (PWM)Motor A speed
ENB7 (PWM)Motor B speed
Arduino C++ — Bluetooth Car
#include <SoftwareSerial.h>
SoftwareSerial btSerial(10, 11);

// Motor pins
const int IN1=2, IN2=3, IN3=4, IN4=5, ENA=6, ENB=7;
int spd = 180;  // 0-255

void setup() {
  btSerial.begin(9600);
  int pins[] = {IN1,IN2,IN3,IN4,ENA,ENB};
  for(int p : pins) pinMode(p, OUTPUT);
  analogWrite(ENA, spd);
  analogWrite(ENB, spd);
}

void loop() {
  if (btSerial.available()) {
    char c = btSerial.read();
    switch(c) {
      case 'F': forward();  break;
      case 'B': backward(); break;
      case 'L': turnLeft(); break;
      case 'R': turnRight(); break;
      case 'S': stopCar();  break;
    }
  }
}

void forward()  { digitalWrite(IN1,HIGH); digitalWrite(IN2,LOW); digitalWrite(IN3,HIGH); digitalWrite(IN4,LOW); }
void backward() { digitalWrite(IN1,LOW);  digitalWrite(IN2,HIGH); digitalWrite(IN3,LOW);  digitalWrite(IN4,HIGH); }
void turnLeft() { digitalWrite(IN1,LOW);  digitalWrite(IN2,HIGH); digitalWrite(IN3,HIGH); digitalWrite(IN4,LOW); }
void turnRight(){ digitalWrite(IN1,HIGH); digitalWrite(IN2,LOW);  digitalWrite(IN3,LOW);  digitalWrite(IN4,HIGH); }
void stopCar()  { digitalWrite(IN1,LOW);  digitalWrite(IN2,LOW);  digitalWrite(IN3,LOW);  digitalWrite(IN4,LOW); }
