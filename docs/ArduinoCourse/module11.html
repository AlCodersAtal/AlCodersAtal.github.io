Lesson 1 of 5

Arduino Memory Types

The Arduino Uno has three distinct types of memory, each serving a different purpose. Understanding them is crucial for writing efficient, bug-free programs — especially when you start running low on space.

MemorySize (Uno)ContentsPersists?
Flash32 KBYour compiled sketch codeYes (until re-flashed)
SRAM2 KBVariables, stack, heap, at runtimeNo (lost on reset)
EEPROM1 KBSmall permanent storageYes (100,000 write cycles)

The SRAM Problem

2 KB sounds tiny — and it is. String constants, arrays, and String objects all eat SRAM. Running out causes random crashes, resets, and very strange behavior. Use the F() macro to move string literals to Flash:

Arduino C++ — Saving SRAM with F() macro
// BAD — strings stored in SRAM (wastes precious 2KB)
Serial.println("Welcome to AlCoders Arduino Course");

// GOOD — F() stores strings in Flash instead
Serial.println(F("Welcome to AlCoders Arduino Course"));

// Check free SRAM at runtime (useful for debugging)
int freeMemory() {
  extern int __heap_start, *__brkval;
  int v;
  return (int) &v - (__brkval == 0 ? (int) &__heap_start : (int) __brkval);
}

void setup() {
  Serial.begin(9600);
  Serial.print(F("Free SRAM: "));
  Serial.println(freeMemory());
}
⚠️
Avoid the String class The Arduino String class causes heap fragmentation and memory leaks over time. Use character arrays (char[]) and strcpy()/sprintf() instead for robust code.

EEPROM Read & Write

EEPROM (Electrically Erasable Programmable Read-Only Memory) survives power-off, resets, and even reprogramming. It's perfect for storing settings, calibration values, or user preferences.

⚠️
EEPROM Endurance Each EEPROM address can only be written ~100,000 times before it wears out. Use EEPROM.update() instead of EEPROM.write() — it only writes if the value has changed, extending EEPROM life dramatically.
Arduino C++ — EEPROM: Save & load settings
#include <EEPROM.h>

void setup() {
  Serial.begin(9600);

  // Write a byte (0-255) to address 0
  EEPROM.update(0, 42);   // Only writes if value changed

  // Read it back
  byte val = EEPROM.read(0);
  Serial.println(val);  // 42

  // Store a float (uses 4 addresses)
  float temp = 23.5;
  EEPROM.put(10, temp);   // address 10-13

  // Read the float back
  float loadedTemp;
  EEPROM.get(10, loadedTemp);
  Serial.println(loadedTemp);   // 23.50
}

void loop() {}

Storing a Struct

Arduino C++ — Store multiple settings as struct
#include <EEPROM.h>

struct Settings {
  byte brightness;    // 0-255
  int  threshold;     // sensor trigger level
  bool alarmEnabled;
};

Settings cfg = { 128, 500, true };

void saveSettings() { EEPROM.put(0, cfg); }
void loadSettings() { EEPROM.get(0, cfg); }

SD Card Module

When you need to store kilobytes or megabytes of data (sensor logs, images, config files), an SD card module is your answer. It communicates via SPI and uses the built-in SD library.

Wiring (SPI)

SD Module PinArduino Uno Pin
VCC5V (some modules need 3.3V — check!)
GNDGND
MOSI11
MISO12
SCK13
CS10 (or any digital pin)
Arduino C++ — Write sensor data to SD card
#include <SPI.h>
#include <SD.h>

const int CS_PIN = 10;

void setup() {
  Serial.begin(9600);
  if (!SD.begin(CS_PIN)) {
    Serial.println(F("SD card init failed!"));
    return;
  }
  Serial.println(F("SD card ready."));
}

void logData(float temp, int light) {
  File f = SD.open("data.csv", FILE_WRITE);
  if (f) {
    f.print(millis());  f.print(",");
    f.print(temp);     f.print(",");
    f.println(light);
    f.close();   // ALWAYS close! Or data may be lost
  }
}

void loop() {
  float temperature = analogRead(A0) * 0.488;
  int   lightLevel  = analogRead(A1);
  logData(temperature, lightLevel);
  delay(5000);   // Log every 5 seconds
}
💡
CSV Format Save data as CSV (comma-separated values) — you can open it directly in Excel or Google Sheets to graph your sensor data. Include headers on first run: "time,temperature,light"

Writing Custom Libraries

A library is just reusable code packaged into a header file (.h) and source file (.cpp). Once you understand how libraries work, you can write your own — making your sketches cleaner and your code portable across projects.

Library Structure

A library lives in Documents/Arduino/libraries/MyLibName/ and contains:

FilePurpose
MyLibName.hClass declaration — lists all functions and variables
MyLibName.cppFunction implementations (the actual code)
keywords.txtIDE syntax highlighting definitions
library.propertiesMetadata (name, version, author)

Example: LED Blinker Library

Blinker.h — Header file
#ifndef BLINKER_H
#define BLINKER_H

#include <Arduino.h>

class Blinker {
  public:
    Blinker(int pin, long interval);
    void update();     // Call in loop()
    void setInterval(long interval);
  private:
    int  _pin;
    long _interval;
    bool _state;
    unsigned long _lastToggle;
};

#endif
Blinker.cpp — Implementation
#include "Blinker.h"

Blinker::Blinker(int pin, long interval) {
  _pin = pin;
  _interval = interval;
  _state = false;
  _lastToggle = 0;
  pinMode(_pin, OUTPUT);
}

void Blinker::update() {
  unsigned long now = millis();
  if (now - _lastToggle >= _interval) {
    _lastToggle = now;
    _state = !_state;
    digitalWrite(_pin, _state);
  }
}

void Blinker::setInterval(long interval) {
  _interval = interval;
}
Sketch — Using your custom library
#include <Blinker.h>

Blinker led1(13, 500);   // pin 13, blink every 500ms
Blinker led2(12, 200);   // pin 12, blink every 200ms

void setup() {}

void loop() {
  led1.update();
  led2.update();
  // Both LEDs blink independently with no delay()
}

Data Logger Project

Build a complete Environmental Data Logger that reads temperature + humidity, timestamps each reading using a DS3231 RTC (Real-Time Clock), saves to SD card in CSV format, and stores the last reading to EEPROM for safe recovery.

Hardware Required

ComponentPurpose
Arduino UnoMain controller
DHT22 sensorTemperature + Humidity
DS3231 RTC moduleReal timestamps (I2C)
SD card moduleLog storage (SPI)
16x2 LCD (I2C)Display current readings
Arduino C++ — Complete Data Logger
#include <DHT.h>
#include <RTClib.h>
#include <SD.h>
#include <SPI.h>
#include <EEPROM.h>

DHT dht(7, DHT22);
RTC_DS3231 rtc;

void setup() {
  Serial.begin(9600);
  dht.begin();
  rtc.begin();
  SD.begin(10);

  // Write CSV header if file doesn't exist
  if (!SD.exists("log.csv")) {
    File f = SD.open("log.csv", FILE_WRITE);
    f.println("datetime,temp_c,humidity_pct");
    f.close();
  }
}

void loop() {
  DateTime now = rtc.now();
  float temp = dht.readTemperature();
  float hum  = dht.readHumidity();

  if (!isnan(temp)) {
    // Save to SD card
    File f = SD.open("log.csv", FILE_WRITE);
    f.print(now.timestamp(DateTime::TIMESTAMP_FULL));
    f.print(","); f.print(temp);
    f.print(","); f.println(hum);
    f.close();

    // Back up last temp to EEPROM (address 0)
    EEPROM.put(0, temp);
    
    Serial.print(temp); Serial.print("°C  ");
    Serial.print(hum);  Serial.println("%");
  }

  delay(60000);  // Log every minute
}
