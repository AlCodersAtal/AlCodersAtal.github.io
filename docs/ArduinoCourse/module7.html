Communication Protocols

Learn how Arduino talks to other devices. Master Serial for debugging and PC communication, I2C for connecting multiple sensors on 2 wires, and SPI for high-speed data transfers.

⏱️ ~2 hours
📗 5 lessons
🟡 Intermediate

💬 Lesson 1: Serial Communication

You've already used Serial.println() to print debug messages. But Serial is far more powerful — it lets Arduino communicate bidirectionally with your computer, other Arduinos, Raspberry Pis, and any device with a UART interface.

How Serial Works

Serial (UART) sends data one bit at a time. Both sides must agree on a baud rate — the speed in bits per second. Common rates: 9600, 115200. Arduino Uno uses pins TX (pin 1) and RX (pin 0) for hardware serial, though these are shared with USB.

⚠️
Important: Avoid using pins 0 (RX) and 1 (TX) for your own components while uploading sketches — it conflicts with the USB serial connection. Use SoftwareSerial for additional serial ports.

Sending Data from Arduino to PC

Arduino C++
void setup() {
  Serial.begin(9600);   // Match this baud rate in Serial Monitor
}

void loop() {
  int lightLevel = analogRead(A0);
  float voltage = lightLevel * (5.0 / 1023.0);

  Serial.print("Light: ");
  Serial.print(lightLevel);
  Serial.print(" | Voltage: ");
  Serial.print(voltage, 2);  // 2 decimal places
  Serial.println("V");      // println adds newline

  delay(500);
}

Receiving Commands from PC

Arduino C++
#define LED_PIN 13

void setup() {
  Serial.begin(9600);
  pinMode(LED_PIN, OUTPUT);
  Serial.println("Ready. Type 'on' or 'off'");
}

void loop() {
  if (Serial.available() > 0) {
    String input = Serial.readStringUntil('\n');
    input.trim();   // Remove whitespace/newlines
    input.toLowerCase(); // Case-insensitive

    if (input == "on") {
      digitalWrite(LED_PIN, HIGH);
      Serial.println("LED turned ON");
    } else if (input == "off") {
      digitalWrite(LED_PIN, LOW);
      Serial.println("LED turned OFF");
    } else {
      Serial.print("Unknown command: ");
      Serial.println(input);
    }
  }
}

Two Arduinos Talking via SoftwareSerial

Arduino C++
#include <SoftwareSerial.h>

// Create extra serial port on pins 10 (RX) and 11 (TX)
SoftwareSerial mySerial(10, 11);

void setup() {
  Serial.begin(9600);     // For debugging via USB
  mySerial.begin(9600);  // For talking to second Arduino
}

void loop() {
  // Forward from USB to second Arduino
  if (Serial.available())    mySerial.write(Serial.read());
  // Forward from second Arduino to USB
  if (mySerial.available()) Serial.write(mySerial.read());
}

🔗 Lesson 2: I2C Protocol

I2C (Inter-Integrated Circuit) lets you connect up to 127 devices using only 2 wires: SDA (data) and SCL (clock). Each device has a unique address (like 0x27 for LCD, 0x3C for OLED). This is why all our I2C modules connected to A4 and A5.

I2C Key Facts

PropertyValue
Wires needed2 (SDA + SCL) + power
Arduino Uno pinsSDA = A4, SCL = A5
Max devices127 (each needs unique address)
Speed100kHz standard, 400kHz fast
Pull-up resistors4.7kΩ on SDA and SCL (most modules include these)
Best forSensors, displays, EEPROMs — low-speed, short cables

Common I2C Sensors & Their Addresses

DeviceDefault AddressPurpose
LCD 16x2 (PCF8574)0x27 or 0x3FCharacter display
OLED SSD13060x3C or 0x3DPixel display
MPU60500x68 or 0x69Accelerometer + Gyroscope
BMP180/BMP2800x76 or 0x77Pressure + Temperature
DS3231 RTC0x57, 0x68Real-time Clock
ADS11150x48–0x4B16-bit ADC

MPU6050 Accelerometer Example

The MPU6050 is a 6-axis accelerometer/gyroscope combo module. It reads motion on all 3 axes:

Arduino C++
// Install: "MPU6050" by Electronic Cats (Library Manager)
#include <Wire.h>
#include <MPU6050.h>

MPU6050 mpu;

void setup() {
  Serial.begin(9600);
  Wire.begin();
  mpu.initialize();
  Serial.println(mpu.testConnection() ? "MPU6050 OK" : "Connection failed");
}

void loop() {
  int16_t ax, ay, az, gx, gy, gz;
  mpu.getMotion6(&ax, &ay, &az, &gx, &gy, &gz);

  Serial.print("AccX:"); Serial.print(ax / 16384.0, 2);
  Serial.print(" AccY:"); Serial.print(ay / 16384.0, 2);
  Serial.print(" AccZ:"); Serial.println(az / 16384.0, 2);
  delay(200);
}

⚡ Lesson 3: SPI Protocol

SPI (Serial Peripheral Interface) is faster than I2C and ideal for high-speed devices like SD cards, displays (TFT), and wireless modules. It uses 4 wires and a master-slave architecture.

SPI Pins on Arduino Uno

SPI SignalArduino Uno PinPurpose
MOSIPin 11Master Out, Slave In (data to device)
MISOPin 12Master In, Slave Out (data from device)
SCKPin 13Clock signal
CS/SSPin 10 (any digital)Chip Select — activates specific device

I2C vs SPI Comparison

FeatureI2CSPI
Wires2 (SDA, SCL)4 (MOSI, MISO, SCK, CS)
Speed100–400 kHzUp to 80 MHz
Devices127 (by address)Many (one CS pin each)
Best forSensors, slow peripheralsSD cards, displays, radio modules

SPI Example: MAX7219 LED Matrix

Arduino C++
// Install "LedControl" library by Eberhard Fahle
#include <LedControl.h>

// DIN=11, CLK=13, CS=10, number of matrices=1
LedControl lc(11, 13, 10, 1);

void setup() {
  lc.shutdown(0, false);  // Wake up MAX7219
  lc.setIntensity(0, 8); // Brightness 0–15
  lc.clearDisplay(0);
}

void loop() {
  // Set individual LEDs: row, col, state
  for (int r = 0; r < 8; r++) {
    for (int c = 0; c < 8; c++) {
      lc.setLed(0, r, c, true);
      delay(30);
    }
  }
  lc.clearDisplay(0);
  delay(500);
}

🔌 Lesson 4: Connecting Multiple Devices

Real projects use multiple devices at once. Here's how to combine I2C devices on the same bus without conflict, and manage addressing properly.

Multiple I2C Devices — No Conflict

All I2C devices share the same SDA and SCL wires. They co-exist because each has a unique address. Wire all VCCs to 5V, all GNDs to GND, all SDAs to A4, all SCLs to A5:

Arduino C++
// OLED + LCD + BMP280 all on I2C simultaneously
#include <Wire.h>
#include <Adafruit_SSD1306.h>
#include <LiquidCrystal_I2C.h>
#include <Adafruit_BMP280.h>

Adafruit_SSD1306 oled(128, 64, &Wire, -1);
LiquidCrystal_I2C lcd(0x27, 16, 2);
Adafruit_BMP280 bmp;

void setup() {
  Wire.begin();
  oled.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  lcd.init(); lcd.backlight();
  bmp.begin(0x76);
}

void loop() {
  float temp = bmp.readTemperature();
  float pres = bmp.readPressure() / 100.0; // hPa

  // Show on LCD
  lcd.setCursor(0, 0);
  lcd.print("Temp: "); lcd.print(temp, 1); lcd.print("C");

  // Show on OLED
  oled.clearDisplay();
  oled.setCursor(0, 0);
  oled.print("Pressure: "); oled.print(pres);
  oled.display();
  delay(1000);
}
✅
Address Conflict? If two devices share the same address (some modules let you change it via solder pads), you can use a TCA9548A I2C multiplexer to isolate them on separate channels — up to 8 same-address devices on one bus!

📈 Lesson 5: Real-Time Serial Plotter

Arduino IDE has a built-in Serial Plotter that graphs values in real time. Just go to Tools → Serial Plotter. Print comma-separated values and it graphs multiple lines simultaneously.

Arduino C++
// Serial Plotter — graph 3 sensor values live
void setup() {
  Serial.begin(9600);
}

void loop() {
  int sensor1 = analogRead(A0);
  int sensor2 = analogRead(A1);
  int sensor3 = analogRead(A2);

  // Print comma-separated — each becomes one plot line
  Serial.print(sensor1);
  Serial.print(",");
  Serial.print(sensor2);
  Serial.print(",");
  Serial.println(sensor3); // println ends the row
  delay(50); // ~20 samples per second
}
💡
Tip: Add labels like Serial.print("Light:"); before the value and the plotter will label the graph line for you (Arduino IDE 2.x only).
