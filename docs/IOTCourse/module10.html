Project: Smart Home Dashboard

Build a multi-room smart home system: multiple ESP32 nodes send sensor data via MQTT to a Raspberry Pi hub, which runs Node-RED and displays a live dashboard with room temperatures, motion events, and device controls.

System Architecture

  • ESP32 Node A (bedroom) — DHT11 temperature/humidity, publishes to home/bedroom/temp
  • ESP32 Node B (living room) — PIR motion + DHT11, publishes to home/living/motion and home/living/temp
  • Raspberry Pi — runs Mosquitto MQTT broker + Node-RED
  • Node-RED Dashboard — charts, gauges, event log at pi-ip:1880/ui

Multi-Room ESP32 Node Code

Arduino (ESP32 — Room Node)
#include <WiFi.h> #include <PubSubClient.h> #include <ArduinoJson.h> #include <DHT.h> const char* mqttBroker = "192.168.1.100"; // Raspberry Pi IP const char* roomId = "bedroom"; // change per node DHT dht(4, DHT11); WiFiClient wc; PubSubClient mqtt(wc); void setup() { dht.begin(); WiFi.begin("SSID", "PASS"); while (WiFi.status() != WL_CONNECTED) delay(500); mqtt.setServer(mqttBroker, 1883); mqtt.connect(roomId); // unique client ID per node } void publishSensors() { StaticJsonDocument<128> doc; doc["room"] = roomId; doc["temp"] = dht.readTemperature(); doc["humidity"] = dht.readHumidity(); doc["uptime"] = millis() / 1000; char buf[128]; serializeJson(doc, buf); String topic = String("home/") + roomId + "/sensors"; mqtt.publish(topic.c_str(), buf); } void loop() { if (!mqtt.connected()) mqtt.connect(roomId); mqtt.loop(); publishSensors(); delay(30000); }

Node-RED Flow Setup

  1. Install Mosquitto on Raspberry Pi: sudo apt install mosquitto mosquitto-clients
  2. Install Node-RED: bash <(curl -sL https://raw.githubusercontent.com/node-red/linux-installers/master/deb/update-nodejs-and-nodered)
  3. Install dashboard palette: node-red-dashboard from the palette manager
  4. Add MQTT-in node → JSON node → Chart node for each room
  5. Add a Text node for "Last motion detected" timestamps
  6. Deploy and open http://raspberry-pi-ip:1880/ui
Unique MQTT Client IDs
Each ESP32 node must connect with a unique client ID (e.g., "bedroom", "living-room"). If two devices use the same ID, one will kick the other off the broker.

Project: GPS Asset Tracker

Track the real-time location of a vehicle, bag, or pet using a GPS module + ESP32. Location is sent via MQTT or HTTP and plotted on a live Google Maps-powered web page.

GPS Parsing + Cloud Upload Code

Arduino (ESP32)
#include <TinyGPS++.h> #include <WiFi.h> #include <HTTPClient.h> TinyGPSPlus gps; HardwareSerial gpsSerial(2); // UART2 void setup() { Serial.begin(115200); gpsSerial.begin(9600, SERIAL_8N1, 16, 17); WiFi.begin("SSID", "PASS"); while (WiFi.status() != WL_CONNECTED) delay(500); } void sendLocation(double lat, double lng) { HTTPClient http; String url = String("https://api.thingspeak.com/update?api_key=YOUR_KEY") + "&field1=" + lat + "&field2=" + lng; http.begin(url); http.GET(); http.end(); } void loop() { while (gpsSerial.available()) gps.encode(gpsSerial.read()); if (gps.location.isUpdated() && gps.location.isValid()) { double lat = gps.location.lat(); double lng = gps.location.lng(); Serial.printf("Lat: %.6f, Lng: %.6f\n", lat, lng); sendLocation(lat, lng); delay(15000); // send every 15 seconds } }
First GPS Fix Takes Time
Cold start (no previous fix) can take 1–5 minutes outdoors with clear sky view. Once a fix is acquired ("hot start"), subsequent fixes are much faster. The GPS LED blinks once per second when a fix is acquired.

Project: Voice-Controlled IoT Devices

Control your IoT devices with voice commands using Google Assistant and IFTTT. Say "Hey Google, turn on the fan" — and your ESP32 switches a relay.

How the Chain Works

  • You say a phrase to Google Assistant
  • Google Assistant triggers an IFTTT Applet
  • IFTTT publishes a message to an Adafruit IO feed via Webhooks
  • ESP32 subscribed to that feed receives the command and acts

The total latency from voice to device is typically 1–3 seconds — good enough for home automation.

IFTTT Applet Setup

  1. Create Applet: IF Google Assistant → "Say a phrase with a text ingredient"
  2. Phrase: "Turn $ the fan" ($ = on/off)
  3. THEN Adafruit IO → Set feed value
  4. Feed: fan-control, Value: {{TextField}}
  5. Repeat for lights, pump, etc.

ESP32 — Subscribe and Control Relay

Arduino (ESP32)
#include <WiFi.h> #include <Adafruit_MQTT.h> #include <Adafruit_MQTT_Client.h> #define RELAY_PIN 26 #define AIO_USER "your_username" #define AIO_KEY "your_aio_key" WiFiClient wc; Adafruit_MQTT_Client mqtt(&wc, "io.adafruit.com", 1883, AIO_USER, AIO_KEY); Adafruit_MQTT_Subscribe fanFeed(&mqtt, AIO_USER "/feeds/fan-control"); void setup() { pinMode(RELAY_PIN, OUTPUT); digitalWrite(RELAY_PIN, LOW); // relay off by default WiFi.begin("SSID", "PASS"); while (WiFi.status() != WL_CONNECTED) delay(500); mqtt.subscribe(&fanFeed); mqtt.connect(); } void loop() { if (!mqtt.connected()) mqtt.connect(); Adafruit_MQTT_Subscribe* sub = mqtt.readSubscription(500); if (sub == &fanFeed) { String cmd = (char*)fanFeed.lastread; cmd.toLowerCase(); if (cmd == "on") { digitalWrite(RELAY_PIN, HIGH); Serial.println("Fan ON"); } else if (cmd == "off") { digitalWrite(RELAY_PIN, LOW); Serial.println("Fan OFF"); } } }
Relay Safety
Relays switch mains voltage (220V in India). Always use a pre-built relay module with optocoupler isolation. Never touch relay terminals when the circuit is live. For classroom projects, only switch 5V/12V DC loads.

Project: Industrial Environment Monitor

Simulate an industrial IoT node: an ESP32 monitors temperature, humidity, gas levels, and machine vibration, publishes to AWS IoT Core via MQTT over TLS, and triggers SMS alerts when thresholds are crossed.

Sensor Suite

BME280
Temperature, humidity, pressure — I²C
MQ-2 Gas Sensor
LPG, smoke, CO detection — analog
MPU-6050 IMU
Accelerometer + gyroscope for vibration — I²C
Buzzer
Local alarm — GPIO digital output

Multi-Sensor JSON Payload

Arduino (ESP32) — Payload Builder
#include <ArduinoJson.h> #include <Adafruit_BME280.h> #include <Adafruit_MPU6050.h> Adafruit_BME280 bme; Adafruit_MPU6050 mpu; void buildAndPublish(PubSubClient& mqtt) { sensors_event_t a, g, temp; mpu.getEvent(&a, &g, &temp); StaticJsonDocument<256> doc; doc["device"] = "factory-node-01"; doc["temp"] = bme.readTemperature(); doc["humidity"] = bme.readHumidity(); doc["pressure"] = bme.readPressure() / 100.0; doc["gas_raw"] = analogRead(34); doc["accel_x"] = a.acceleration.x; doc["accel_y"] = a.acceleration.y; doc["accel_z"] = a.acceleration.z; doc["timestamp"] = millis(); char payload[256]; serializeJson(doc, payload); mqtt.publish("factory/sensor/data", payload); // Local alarm for high gas levels if (doc["gas_raw"].as<int>() > 1800) { digitalWrite(25, HIGH); // buzzer on delay(2000); digitalWrite(25, LOW); } }

AWS IoT Rules Engine

Once data arrives in AWS IoT Core, a Rules Engine SQL query can extract values and route them:

AWS IoT Rule SQL
-- Route all sensor data to DynamoDB SELECT * FROM 'factory/sensor/data' -- Trigger SNS alert only when gas is high SELECT device, gas_raw, timestamp FROM 'factory/sensor/data' WHERE gas_raw > 1800

Capstone: Design Your Own IoT System

Congratulations — you've reached the end of the AlCoders IoT course! This final lesson is your capstone: design and build a complete IoT solution of your own, from problem statement to working prototype.

The IoT Design Process

  1. Define the problem — What real-world problem are you solving? Who benefits?
  2. Identify sensors & actuators — What data do you need? What actions should the system take?
  3. Choose your platform — Cloud or local? MQTT or HTTP? Which dashboard?
  4. Design the data model — What JSON fields will you publish? What topics?
  5. Build incrementally — Get sensor reading working first. Add cloud. Add dashboard. Add alerts.
  6. Test edge cases — What happens when Wi-Fi drops? When sensors fail?
  7. Present and document — Write a README. Make a wiring diagram. Demo it!
