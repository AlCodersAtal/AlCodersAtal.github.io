Microcontrollers vs Microprocessors

Before picking a board, understand the fundamental difference between a microcontroller (MCU) and a microprocessor (MPU). This shapes everything in IoT hardware design.

Microcontroller (MCU)

A microcontroller is a complete computer on a single chip. It contains:

  • CPU – executes your code
  • RAM – temporary storage (a few KB to a few MB)
  • Flash memory – stores your program permanently
  • I/O pins – connect to sensors, LEDs, motors
  • Peripherals – built-in UART, SPI, I2C, ADC, PWM

MCUs are designed to run one specific program reliably, with no operating system (or a tiny RTOS). They start instantly and run for years on batteries.

Examples
ATmega328P (Arduino Uno), ESP8266, ESP32, STM32, ATtiny

Microprocessor (MPU)

A microprocessor is just the CPU — it needs external RAM, storage, and peripherals. It runs a full operating system (Linux, Windows) and can do many things at once. Much more powerful, but also uses much more power and costs more.

Examples
Raspberry Pi (BCM2711), Nvidia Jetson, BeagleBone Black

Comparison Table

FeatureMicrocontrollerMicroprocessor

🎯 IoT Choice
Most IoT edge devices use microcontrollers because of low power, low cost, and simplicity. Microprocessors are used when you need heavy processing (AI, video analysis).

Arduino for IoT

Arduino is the most beginner-friendly microcontroller platform in the world. It's an excellent starting point for IoT — even if most production IoT uses ESP32 or other Wi-Fi boards.

What is Arduino?

Arduino is an open-source electronics platform combining a microcontroller board and a software IDE. The Arduino Uno uses the ATmega328P chip running at 16 MHz with 32 KB of flash and 2 KB of RAM.

Arduino's real power is its ecosystem: thousands of libraries, tutorials, and community projects. The Arduino IDE lets you write and upload code with one click.

Your First Arduino Sketch

Arduino C
// Blink an LED – the "Hello World" of Arduino void setup() { // Pin 13 has a built-in LED on Arduino Uno pinMode(13, OUTPUT); } void loop() { digitalWrite(13, HIGH); // LED on delay(1000); // wait 1 second digitalWrite(13, LOW); // LED off delay(1000); // wait 1 second }

Every Arduino sketch has two functions: setup() runs once at startup, loop() runs forever in a continuous loop.

Arduino for IoT — The Limitation

Arduino Uno has no built-in Wi-Fi or Bluetooth. To add internet connectivity you need an extra shield (like ESP8266 module). This is why most IoT projects quickly move to the ESP8266 or ESP32, which have Wi-Fi built in.

⚠️ Learn Arduino First
Even if you'll use ESP32 in production, learn Arduino first — the programming concepts, wiring skills, and debugging habits transfer directly.

ESP8266 & ESP32 — The Wi-Fi Boards

The ESP8266 and ESP32 by Espressif are the go-to chips for IoT. They combine a capable processor with built-in Wi-Fi (and Bluetooth on ESP32) at an incredibly low price.

ESP8266 — The Wi-Fi Pioneer

Released in 2014, the ESP8266 changed IoT forever. For under ₹100, you get:

  • 32-bit Tensilica L106 CPU @ 80/160 MHz
  • Built-in Wi-Fi 802.11 b/g/n
  • 80 KB RAM, 4 MB flash (on NodeMCU boards)
  • GPIO, UART, SPI, I2C, ADC (1 channel)
  • Programmable with Arduino IDE

Most popular form: NodeMCU v1/v2 — a breakout board with USB and all pins exposed.

ESP32 — The IoT Powerhouse

The ESP32 is the upgrade. For ₹250–₹400 you get significantly more capability:

  • Dual-core 240 MHz Xtensa LX6 CPU
  • Wi-Fi 802.11 b/g/n AND Bluetooth 4.2 / BLE
  • 520 KB RAM, 4 MB flash (typical)
  • 34 GPIO pins, 18-channel ADC, 2× DAC
  • Built-in Hall sensor, temperature sensor
  • Supports deep sleep: draw as little as 10 µA
  • Hardware encryption/decryption acceleration
💡 Use ESP32
For any new IoT project, default to ESP32. It's powerful enough for everything in this course, and only costs a little more than ESP8266.

Raspberry Pi in IoT

Raspberry Pi is a credit-card-sized single-board computer that runs Linux. It's the powerhouse option for IoT projects that need AI, video, or a web server.

What is Raspberry Pi?

Raspberry Pi runs a full Linux operating system. Unlike Arduino or ESP32, it can run Python, Node.js, databases, web servers — anything a regular computer can do. It's perfect when your IoT project needs heavy lifting.

  • Raspberry Pi 4: Quad-core ARM Cortex-A72 @ 1.8 GHz, 1–8 GB RAM
  • Wi-Fi, Bluetooth, Gigabit Ethernet, USB 3.0
  • HDMI output — connect a monitor directly
  • 40-pin GPIO header — connect sensors just like Arduino
  • Runs Raspberry Pi OS (Linux) or Ubuntu

Reading a Sensor with Python on Pi

Python 3
import RPi.GPIO as GPIO import Adafruit_DHT import time SENSOR = Adafruit_DHT.DHT11 PIN = 4 # GPIO pin number while True: humidity, temperature = Adafruit_DHT.read_retry(SENSOR, PIN) if humidity is not None: print(f"Temp: {temperature}°C Humidity: {humidity}%") else: print("Sensor read failed") time.sleep(2)

Choosing the Right Board

With so many options, how do you pick the right board? This lesson gives you a decision framework for every IoT project.

The Decision Framework

Ask these questions in order:

  1. Is it battery-powered? → Yes: MCU (ESP32 with deep sleep). No: Pi is fine.
  2. Does it need Wi-Fi? → Yes: ESP8266/ESP32. No: Arduino Uno is enough.
  3. Does it need Bluetooth too? → Yes: ESP32.
  4. Does it need Linux / Python? → Yes: Raspberry Pi.
  5. Does it need video/AI? → Yes: Raspberry Pi 4 or Jetson.
  6. Ultra-low cost (< ₹50/unit)? → Yes: ATtiny or bare ESP8266 chip.
