Storing IoT Data

Sensor readings are only useful if you can store and retrieve them. Different storage technologies suit different IoT use-cases — from a simple CSV on an SD card to a managed cloud time-series database.

Storage Options

SD Card (local)
Cheapest. No internet needed. Good for data logging in remote locations. Limited by card size.
SPIFFS / LittleFS
ESP32's built-in flash filesystem. Store config files or small logs. Data lost if firmware erased.
Firebase Realtime DB
JSON tree synced in real time. Easy to query from web apps. Free tier: 1 GB storage, 10 GB/month transfer.
InfluxDB
Purpose-built time-series database. Extremely fast for sensor data. Can run locally or on cloud.

Logging to SD Card from ESP32

Arduino (ESP32)
#include <SD.h> #include <SPI.h> #define SD_CS 5 // chip select pin void setup() { Serial.begin(115200); SD.begin(SD_CS); } void logData(float temp, float humidity) { File f = SD.open("/log.csv", FILE_APPEND); if (f) { f.print(millis()); f.print(","); f.print(temp); f.print(","); f.println(humidity); f.close(); } } void loop() { logData(28.4, 65.2); // replace with real sensor reads delay(60000); // log every 1 minute }

Node-RED: Visual IoT Programming

Node-RED is a browser-based flow programming tool. You connect nodes (MQTT in → function → chart) visually instead of writing code. It runs on Raspberry Pi or any computer.

  • Drag an MQTT in node and configure your broker
  • Add a JSON node to parse incoming payloads
  • Connect to a chart node from the Dashboard palette
  • Click Deploy — your dashboard is live at localhost:1880/ui

Building a Simple Web Dashboard (HTML + JS)

You can also build a live dashboard yourself using the Fetch API to pull data from ThingSpeak:

JavaScript
const channelId = "YOUR_CHANNEL_ID"; const apiKey = "YOUR_READ_KEY"; async function fetchLatest() { const res = await fetch( `https://api.thingspeak.com/channels/${channelId}/feeds/last.json?api_key=${apiKey}` ); const data = await res.json(); document.getElementById('temp').textContent = data.field1 + ' °C'; document.getElementById('hum').textContent = data.field2 + ' %'; } fetchLatest(); setInterval(fetchLatest, 15000); // refresh every 15 s
Chart.js for Beautiful Graphs
Pair the Fetch API with Chart.js (chartjs.org) to build professional line charts in your browser — 20 lines of JavaScript and you have a live-updating graph.

Alerts & Notifications

An IoT system isn't very useful if you have to watch a dashboard 24/7. Alerts let the system notify you the moment something goes wrong — temperature spikes, soil dries out, motion detected.

Alert Delivery Methods

Email
Free, reliable, works via Gmail SMTP or services like SendGrid. Best for non-urgent alerts.
SMS / WhatsApp
Immediate attention. Use Twilio (SMS) or Twilio WhatsApp API. Costs per message.
Push Notification
App alerts. Blynk and Adafruit IO have built-in push. Free and fast.
Buzzer / LED
On-device alert — no internet needed. Great for local warnings.

Threshold Alerts on Adafruit IO

  1. Open your feed → click Feed Info
  2. Go to Triggers → New Trigger
  3. Set condition: temperature > 35
  4. Action: Send Email to your address
  5. Save — Adafruit IO will email you whenever the condition is met

Sending Email from ESP32 via Gmail SMTP

Use the ESP Mail Client library (by Mobizt) to send email directly from your microcontroller:

Arduino (ESP32)
#include <ESP_Mail_Client.h> SMTPSession smtp; void sendAlert(float temp) { ESP_Mail_Session session; session.server.host_name = "smtp.gmail.com"; session.server.port = 465; session.login.email = "youremail@gmail.com"; session.login.password = "your_app_password"; // use App Password, not main pass SMTP_Message msg; msg.sender.name = "IoT Alert"; msg.sender.email = "youremail@gmail.com"; msg.subject = "High Temperature Warning!"; msg.addRecipient("Me", "youremail@gmail.com"); msg.text.content = (String("Temp is ") + temp + " C — check your device!").c_str(); smtp.connect(&session); MailClient.sendMail(&smtp, &msg); }
Use Gmail App Passwords
Never use your main Gmail password in code. Enable 2FA on your Google account, then generate an App Password under Security settings. This is a 16-character code that only works for your app.
Why should you NOT use your main Gmail password in ESP32 code?
The ESP32 doesn't support SSL
The code could be read by others, exposing your account credentials
Gmail blocks SMTP from microcontrollers
Main passwords are too long for Arduino strings

Automation with IFTTT

IFTTT (If This Then That) connects hundreds of services together with simple "Applets". IoT devices can trigger Google Sheets updates, Telegram messages, smart lights, and much more — no server required.

How IFTTT Works

Every IFTTT Applet has two parts:

  • Trigger (IF THIS) — an event that starts the chain. For IoT: a Webhooks HTTP request from your device.
  • Action (THEN THAT) — what happens next. Options include: append row to Google Sheet, send Telegram message, turn on Philips Hue lights, post to Twitter, etc.

IFTTT Webhooks from ESP32

The Webhooks service gives you a URL endpoint. When your ESP32 sends an HTTP request to it, IFTTT fires the connected action.

Arduino (ESP32)
#include <WiFi.h> #include <HTTPClient.h> const char* iftttKey = "YOUR_IFTTT_KEY"; const char* eventName = "motion_detected"; void triggerIFTTT(String value1) { HTTPClient http; String url = String("https://maker.ifttt.com/trigger/") + eventName + "/with/key/" + iftttKey; http.begin(url); http.addHeader("Content-Type", "application/json"); String payload = String('{"value1":"') + value1 + '"}'; http.POST(payload); http.end(); } void loop() { int pir = digitalRead(14); if (pir == HIGH) { triggerIFTTT("Motion at front door"); delay(30000); // cool-down: 30s before next trigger } }

Edge Computing in IoT

Edge computing moves processing closer to the device — on the microcontroller itself or a nearby gateway — instead of sending all data to the cloud. This reduces latency, saves bandwidth, and enables offline operation.

Cloud vs Edge

Cloud ProcessingEdge Processing
Latency100–500ms (network round trip)<1ms (local)
BandwidthAll raw data sentOnly results/events sent
Offline?NoYes
Processing powerUnlimitedLimited by hardware
CostData transfer feesHigher hardware cost

On-Device Edge Processing

Instead of sending every sensor reading, only send an alert when something unusual happens. This is edge computing at its simplest:

Arduino (ESP32)
float readings[10]; int idx = 0; void collectAndProcess() { readings[idx++] = readTemperature(); if (idx == 10) { // Compute average at the edge float avg = 0; for (int i = 0; i < 10; i++) avg += readings[i]; avg /= 10; idx = 0; // Only send average, not all 10 raw readings sendToCloud(avg); // Trigger alert locally — no cloud needed if (avg > 38.0) triggerLocalBuzzer(); } }

TinyML: Machine Learning at the Edge

Libraries like TensorFlow Lite for Microcontrollers let you run trained ML models on an ESP32 or Arduino Nano 33 BLE Sense. Real-world uses:

  • Wake word detection — "Hey device!" processed locally, no audio sent to cloud
  • Gesture recognition — IMU data classified on-chip
  • Anomaly detection — motor vibration patterns analysed locally
  • Image classification — ESP32-CAM identifies objects without uploading photos
Edge + Cloud Together
Best IoT systems use both: edge for fast local decisions and offline operation, cloud for long-term storage, trend analysis, and remote management. Think of edge as the first filter that only sends meaningful data upward.
