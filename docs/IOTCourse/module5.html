Arduino IDE Setup for IoT

The Arduino IDE is where you write, compile, and upload code to your board. Setting it up correctly for IoT development — especially for ESP32 — is the first step.

Installing Arduino IDE

  1. Download Arduino IDE 2.x from arduino.cc/en/software
  2. Install for your OS (Windows / macOS / Linux)
  3. Open IDE → File → Preferences
  4. In "Additional Board Manager URLs" add:
    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  5. Tools → Board → Board Manager → search "esp32" → Install "esp32 by Espressif"
  6. Tools → Board → ESP32 Arduino → ESP32 Dev Module

Key Arduino IDE Features for IoT

  • Serial Monitor (Ctrl+Shift+M): See your device's debug output. Set baud rate to match Serial.begin() value.
  • Library Manager (Ctrl+Shift+I): Install sensor and protocol libraries with one click.
  • Board Manager: Add support for non-Arduino boards like ESP32.
  • Boards & Port selector: Pick the right board and COM port before uploading.
🔌 Upload Tip
When uploading to ESP32, hold the BOOT button on the board while clicking Upload. Release when "Connecting..." appears. Some boards (like ESP32 DevKit v1) do this automatically.

Reading Sensor Data in Code

Reading multiple sensors, combining their data into a JSON object, and printing it ready for transmission — this is the core loop of every IoT device.

The IoT Device Main Loop

Every IoT device code follows this pattern:

Arduino C (ESP32)
#include <DHT.h> #include <ArduinoJson.h> DHT dht(4, DHT11); int lightPin = 34; // ADC pin void setup() { Serial.begin(115200); dht.begin(); } String readSensors() { float temp = dht.readTemperature(); float hum = dht.readHumidity(); int light = analogRead(lightPin); // Create JSON document StaticJsonDocument<200> doc; doc["temperature"] = temp; doc["humidity"] = hum; doc["light"] = light; doc["device"] = "AlCoders-Node-1"; String output; serializeJson(doc, output); return output; } void loop() { String json = readSensors(); Serial.println(json); // Next: send json over MQTT or HTTP delay(10000); }

Output: {"temperature":28.5,"humidity":65,"light":2045,"device":"AlCoders-Node-1"}

Sending Data to the Cloud

This is the most important skill in IoT programming — taking sensor readings and reliably pushing them to a cloud service.

Complete ESP32 → Cloud Flow

Arduino C (ESP32) — Full Example
#include <WiFi.h> #include <PubSubClient.h> #include <ArduinoJson.h> #include <DHT.h> // ── Config ────────────────────────────── const char* WIFI_SSID = "YourWiFi"; const char* WIFI_PASS = "YourPassword"; const char* MQTT_BROKER = "broker.hivemq.com"; const char* TOPIC = "alcoders/lab/sensors"; DHT dht(4, DHT11); WiFiClient wifiClient; PubSubClient mqtt(wifiClient); void connectWiFi() { WiFi.begin(WIFI_SSID, WIFI_PASS); while (WiFi.status() != WL_CONNECTED) { delay(500); Serial.print("."); } Serial.println(" WiFi OK"); } void connectMQTT() { mqtt.setServer(MQTT_BROKER, 1883); while (!mqtt.connected()) { mqtt.connect("AlCodersNode"); delay(1000); } Serial.println("MQTT OK"); } void setup() { Serial.begin(115200); dht.begin(); connectWiFi(); connectMQTT(); } void loop() { if (!mqtt.connected()) connectMQTT(); mqtt.loop(); StaticJsonDocument<200> doc; doc["temp"] = dht.readTemperature(); doc["humidity"] = dht.readHumidity(); char buf[200]; serializeJson(doc, buf); mqtt.publish(TOPIC, buf); Serial.println("Published: " + String(buf)); delay(10000); }

Error Handling Best Practices

  • Always check Wi-Fi is connected before sending data
  • Always check sensor readings for NaN before publishing
  • Implement reconnect logic for both Wi-Fi and MQTT
  • Use watchdog timer to reset device if stuck

Receiving Commands from Cloud

IoT is two-way. Not only does your device send data up to the cloud — it also receives commands back, allowing remote control of actuators.

MQTT Subscribe — Receiving Commands

Arduino C (ESP32)
#include <WiFi.h> #include <PubSubClient.h> int relayPin = 26; // Called when a message arrives on subscribed topic void onMessage(char* topic, byte* payload, unsigned int length) { String msg = ""; for (int i = 0; i < length; i++) msg += (char)payload[i]; Serial.print("Command received: "); Serial.println(msg); if (msg == "ON") { digitalWrite(relayPin, LOW); // Activate relay Serial.println("Relay ON"); } else if (msg == "OFF") { digitalWrite(relayPin, HIGH); // Deactivate relay Serial.println("Relay OFF"); } } WiFiClient wClient; PubSubClient mqtt(wClient); void setup() { Serial.begin(115200); pinMode(relayPin, OUTPUT); // ... WiFi connect ... mqtt.setServer("broker.hivemq.com", 1883); mqtt.setCallback(onMessage); mqtt.connect("AlCodersRelay"); mqtt.subscribe("alcoders/lab/relay/control"); Serial.println("Subscribed to relay topic"); } void loop() { mqtt.loop(); }

Now publish ON or OFF to the topic from any MQTT client (MQTT Explorer, Blynk, Node-RED) to control the relay remotely.

MicroPython on ESP32

MicroPython is a lean Python 3 implementation that runs on microcontrollers. For Python fans, it's an alternative to Arduino C for ESP32 IoT projects.

Installing MicroPython on ESP32

  1. Download MicroPython firmware from micropython.org/download/esp32
  2. Install esptool: pip install esptool
  3. Flash firmware: esptool.py --chip esp32 erase_flash then esptool.py write_flash -z 0x1000 esp32-firmware.bin
  4. Use Thonny IDE or WebREPL to write and run Python scripts

MicroPython Wi-Fi + MQTT

MicroPython
import network import time from umqtt.simple import MQTTClient import dht from machine import Pin # Connect to Wi-Fi sta = network.WLAN(network.STA_IF) sta.active(True) sta.connect("YourWiFi", "YourPassword") while not sta.isconnected(): time.sleep(0.5) print("Connected:", sta.ifconfig()) # MQTT setup client = MQTTClient("esp32-py", "broker.hivemq.com") client.connect() # DHT sensor on pin 4 sensor = dht.DHT11(Pin(4)) while True: sensor.measure() temp = sensor.temperature() hum = sensor.humidity() msg = f'{{"temp":{temp},"humidity":{hum}}}' client.publish(b"alcoders/lab/sensors", msg.encode()) print("Published:", msg) time.sleep(10)
