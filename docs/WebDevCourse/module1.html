How the Web Works

Before writing a single line of code, understand what actually happens when someone types a URL and presses Enter.

The Web in a Nutshell

The web is made up of millions of servers (computers that store websites) and clients (your browser). When you visit a site, your browser sends a request to a server, and the server sends back files.

ℹ️ Key Terms
  • Browser — Chrome, Firefox, Safari: reads HTML/CSS/JS files and displays them
  • Server — A computer that stores website files and sends them on request
  • URL — The address of a webpage (e.g. https://google.com)
  • HTTP/HTTPS — The protocol browsers use to communicate with servers

What Happens When You Visit a Website?

  1. You type a URL (e.g. https://alcoders.com) into your browser
  2. Your browser looks up the server address using DNS (like a phone book for websites)
  3. Your browser sends an HTTP request to that server
  4. The server sends back HTML, CSS, and JavaScript files
  5. Your browser reads those files and renders the page you see
💡 Remember This
Every webpage you've ever seen is just HTML, CSS, and JavaScript files that your browser downloaded and displayed. That's it!

Frontend vs Backend

  • Frontend — What you see in the browser: HTML, CSS, JavaScript. This is what this course teaches.
  • Backend — What runs on the server: databases, logic, security. Languages like PHP, Node.js.

Setting Up Your Tools

You only need two things to start building websites: a browser and a code editor. Let's set them up.

What You Need

  • A web browser — Chrome or Firefox recommended (you probably already have one)
  • VS Code — A free, powerful code editor made by Microsoft

Installing VS Code

  1. Go to https://code.visualstudio.com
  2. Click the big blue Download button for your operating system
  3. Run the installer and follow the steps
  4. Open VS Code — you're ready!
💡 Recommended Extensions
  • Live Server — Auto-refreshes your browser every time you save a file
  • Prettier — Automatically formats your code to look clean
Install them from the Extensions panel (left sidebar icon that looks like 4 squares).

Setting Up Your First Project Folder

  1. Create a new folder on your Desktop called my-website
  2. Open VS Code and click File → Open Folder
  3. Select your my-website folder
  4. You now have a project workspace!
⚠️ Naming Files
Always use lowercase letters and hyphens (no spaces!) when naming files and folders. my-website ✅    My Website ❌

Your First HTML File

Time to write your very first HTML. Create a file, add the boilerplate, open it in a browser — and see your first webpage come to life.

What is an HTML File?

An HTML file is just a plain text file with a .html extension. When opened in a browser, it displays as a webpage.

ℹ️ HTML stands for
HyperText Markup Language — "markup" means adding labels (tags) to text to tell the browser how to display it.

The HTML Boilerplate

Every HTML page starts with the same basic structure — the boilerplate:

index.html
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My First Webpage</title> </head> <body> <h1>Hello, World!</h1> <p>My first webpage is alive.</p> </body> </html>
  • <!DOCTYPE html> — tells the browser this is HTML5
  • <html> — root container wrapping everything
  • <head> — settings and info (not shown on screen)
  • <title> — text shown on the browser tab
  • <body> — everything visible on the page goes here

What Are HTML Tags?

Tags are the building blocks of HTML. Every element on a webpage is created by wrapping content in tags.

Anatomy of an HTML Tag

Most elements have an opening tag, content, and a closing tag:

Structure
<!-- Opening Content Closing --> <p> This is text. </p> <h1> Big Heading </h1> <strong> Bold text </strong>
💡 Closing Tags
Always have a / before the tag name: </p> means "end paragraph".

Self-Closing Tags

Some tags don't wrap content — they just insert something on their own:

Self-closing
<br> <!-- line break --> <hr> <!-- horizontal divider line --> <img src="photo.jpg" alt="A photo"> <!-- image -->

Attributes

Tags can carry extra information called attributes, written as name="value" inside the opening tag:

Attributes
<a href="https://google.com">Go to Google</a> <img src="cat.jpg" alt="A cute cat" width="300"> <p class="intro" id="first-para">Hello!</p>
⚠️ Remember
Attribute values must always be wrapped in quotes " ". Forgetting this is one of the most common beginner mistakes!
