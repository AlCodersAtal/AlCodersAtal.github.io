Planning Your Portfolio

Great websites start with a plan, not code. Let's map out your portfolio before touching a keyboard.

What Sections Should a Portfolio Have?

  • Header/Navbar — logo, navigation links
  • Hero Section — your name, title, call-to-action buttons
  • About Me — short bio and skills
  • Projects — cards showing your work
  • Contact — form or links to reach you
  • Footer — copyright

Plan Your Design First

Before coding, sketch on paper or use a tool like Figma (free). Decide:

  1. Color scheme — pick 2-3 colours (primary, accent, background)
  2. Font pairing — one for headings, one for body text
  3. Layout — how many columns? What goes where?
  4. Content — what projects will you showcase?
💡 Start Simple
Your first portfolio doesn't need to be perfect. A clean, simple site beats a complicated broken one every time. You can always improve it later!

Building the HTML Structure

Build the complete semantic HTML skeleton of your portfolio — all sections in the right order.

The Full Portfolio HTML

index.html
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Your Name — Portfolio</title> <link rel="stylesheet" href="style.css"> </head> <body> <header> <nav>...</nav> </header> <section id="hero">...</section> <section id="about">...</section> <section id="projects">...</section> <section id="contact">...</section> <footer>...</footer> </body> </html>

Styling with CSS

Apply all your CSS knowledge to make your portfolio look polished, professional, and uniquely yours.

Portfolio CSS Strategy

Use CSS variables at the top of your stylesheet so you can easily change your whole theme:

style.css
/* CSS Variables — your theme */ :root { --primary: #1a5cff; --dark: #1a1a2e; --light: #f7f7f5; --text: #333; --muted: #888; } * { margin:0; padding:0; box-sizing:border-box; } body { font-family:Arial, sans-serif; color:var(--text); } /* Section padding */ section { padding: 80px 5%; max-width: 1100px; margin: 0 auto; } /* Use your variable */ h1, h2 { color: var(--dark); } .btn { background: var(--primary); }

Module 08 · Lesson 4

Making It Responsive

Your portfolio must work on every screen. Apply mobile-first media queries to every section.

Responsive Portfolio Checklist

  • Viewport meta tag — always include it in <head>
  • Fluid widths — use % and max-width instead of fixed px
  • Stack on mobile — use flex-direction: column on small screens
  • Hide the navbar links — show only the logo on mobile
  • Bigger touch targets — buttons at least 44px tall on mobile

Responsive Projects Grid

Responsive Grid
.projects-grid { display: grid; grid-template-columns: 1fr; /* mobile: 1 col */ gap: 20px; } @media(min-width: 600px) { .projects-grid { grid-template-columns: repeat(2, 1fr); } } @media(min-width: 900px) { .projects-grid { grid-template-columns: repeat(3, 1fr); } }

Publishing Your Site with GitHub Pages

The final step — get your portfolio live on the internet for free using GitHub Pages. No server needed.

What is GitHub Pages?

GitHub Pages is a free hosting service that turns any public GitHub repository into a live website. Perfect for portfolios and static sites.

Step-by-Step Deployment

  1. Create a free account at https://github.com
  2. Click New Repository and name it my-portfolio
  3. Upload your files (index.html, style.css, images folder)
  4. Go to Settings → Pages
  5. Under "Source", select main branch and click Save
  6. Wait 1-2 minutes — your site will be live at:
Your Live URL
https://yourusername.github.io/my-portfolio/
💡 Custom Domain
You can also connect a custom domain (like yourname.com) in the Pages settings for free — you just need to purchase the domain from a registrar like Namecheap or Google Domains.
