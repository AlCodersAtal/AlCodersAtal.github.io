Arduino Uno Car with Motor Driver

A complete step-by-step guide with wiring, code, and explanation

1. Introduction

In this project, we will build a simple Arduino-based robotic car. The car uses DC motors for movement, an L298N Motor Driver for motor control, and Arduino Uno as the brain to send signals. This project forms the foundation for advanced robots like line-following cars, Bluetooth-controlled cars, and obstacle-avoiding cars.

2. Components Required

  • Arduino Uno (x1)
  • L298N Motor Driver Module (x1)
  • DC Motors (x2 or x4) + Wheels
  • Robot chassis (base frame)
  • Battery pack (9V or 12V depending on motors)
  • Jumper wires, breadboard

How it works: The Arduino sends control signals → Motor driver interprets and powers the motors → Car moves.

3. Circuit Connections

  • Arduino Pins 8 & 9 → Motor Driver IN1 & IN2 (Left Motor)
  • Arduino Pins 10 & 11 → Motor Driver IN3 & IN4 (Right Motor)
  • Motor Driver OUT1 & OUT2 → Left Motor terminals
  • Motor Driver OUT3 & OUT4 → Right Motor terminals
  • Battery pack → Motor Driver VCC (12V) and GND
  • Arduino GND → Motor Driver GND (common ground)

Working principle: Each motor has two control pins. Sending HIGH and LOW spins it one way, reversing them spins it the other way. When both motors spin forward → car goes straight.

4. Arduino Code

#include <Arduino.h>

int motor1Pin1 = 8; 
int motor1Pin2 = 9;
int motor2Pin1 = 10; 
int motor2Pin2 = 11;

void setup() {
  pinMode(motor1Pin1, OUTPUT);
  pinMode(motor1Pin2, OUTPUT);
  pinMode(motor2Pin1, OUTPUT);
  pinMode(motor2Pin2, OUTPUT);
}

void loop() {
  moveForward();
  delay(2000);
  moveBackward();
  delay(2000);
  turnLeft();
  delay(2000);
  turnRight();
  delay(2000);
  stopCar();
  delay(2000);
}

void moveForward() {
  digitalWrite(motor1Pin1, HIGH);
  digitalWrite(motor1Pin2, LOW);
  digitalWrite(motor2Pin1, HIGH);
  digitalWrite(motor2Pin2, LOW);
}

void moveBackward() {
  digitalWrite(motor1Pin1, LOW);
  digitalWrite(motor1Pin2, HIGH);
  digitalWrite(motor2Pin1, LOW);
  digitalWrite(motor2Pin2, HIGH);
}

void turnLeft() {
  digitalWrite(motor1Pin1, LOW);
  digitalWrite(motor1Pin2, HIGH);
  digitalWrite(motor2Pin1, HIGH);
  digitalWrite(motor2Pin2, LOW);
}

void turnRight() {
  digitalWrite(motor1Pin1, HIGH);
  digitalWrite(motor1Pin2, LOW);
  digitalWrite(motor2Pin1, LOW);
  digitalWrite(motor2Pin2, HIGH);
}

void stopCar() {
  digitalWrite(motor1Pin1, LOW);
  digitalWrite(motor1Pin2, LOW);
  digitalWrite(motor2Pin1, LOW);
  digitalWrite(motor2Pin2, LOW);
}

Explanation: - moveForward(): Both motors spin forward. - moveBackward(): Both motors spin backward. - turnLeft(): Left motor reverses, right motor moves forward → car rotates left. - turnRight(): Right motor reverses, left motor moves forward → car rotates right. - stopCar(): All motor pins LOW → car halts.

5. Step-by-Step Process

  1. Assemble the chassis and mount the DC motors with wheels.
  2. Connect motors to the L298N motor driver.
  3. Wire the motor driver IN pins to Arduino pins.
  4. Connect battery pack to motor driver for power.
  5. Upload the provided Arduino sketch using Arduino IDE.
  6. Turn on the power → your car should start moving according to the programmed sequence.

Result

Once powered, the Arduino controls the car as follows: - The car moves forward for 2 seconds, - Then backward for 2 seconds, - Turns left for 2 seconds, - Turns right for 2 seconds, - Stops for 2 seconds. This cycle repeats endlessly.

6. Possible Upgrades

  • Ultrasonic Sensor for Obstacle Detection:
    Add an HC-SR04 ultrasonic sensor to the front of your car. Connect the sensor’s trigger and echo pins to Arduino digital pins. In your code, measure the distance to obstacles and automatically stop or turn the car if something is detected within a set range. This upgrade lets your car avoid collisions and navigate around objects.
    Tip: Use the NewPing library for easy distance measurement.
  • Bluetooth Module (HC-05) for Smartphone Control:
    Connect an HC-05 Bluetooth module to your Arduino’s TX/RX pins. Pair the module with your phone and use a Bluetooth controller app to send commands (forward, backward, left, right, stop) to the car. Update your Arduino code to read serial commands and control the motors accordingly. This upgrade allows remote control and interactive fun.
    Tip: Use apps like “Bluetooth RC Car” or “Arduino Bluetooth Controller” for easy setup.
  • Line Following Using IR Sensors:
    Mount two or more IR sensors underneath the car, facing the ground. These sensors detect the contrast between a black line and a white surface. Program your Arduino to adjust motor speeds based on sensor readings, so the car follows the line automatically. This upgrade is great for robotics competitions and learning about feedback systems.
    Tip: Use the QTRSensors library for advanced line tracking.
  • Speed Control with PWM:
    Use Arduino’s analogWrite() function to send PWM signals to the motor driver’s enable pins. This lets you control the speed of each motor for smoother turns and acceleration. You can add speed adjustment features to your remote control or obstacle avoidance logic.
  • App/Web-Based Control:
    Integrate an ESP8266 or ESP32 WiFi module to control your car via a web interface or custom app. Host a simple webpage on the module and send movement commands to the Arduino. This upgrade brings IoT capabilities and remote access from anywhere on your network.
  • Battery Monitoring:
    Add a voltage divider circuit to monitor battery voltage using an analog pin. Display battery status on an LCD or send alerts via Bluetooth/WiFi when the battery is low. This helps prevent unexpected shutdowns and protects your electronics.

Conclusion

Building an Arduino-powered car is a fantastic way to learn about electronics, programming, and robotics. By following this guide, you’ve created a foundation for more advanced projects and upgrades. Whether you add sensors, wireless control, or smart features, each enhancement will deepen your understanding and skills. Keep experimenting, keep learning, and enjoy the journey into robotics!
