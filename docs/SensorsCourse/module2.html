What is Arduino?

Before connecting any sensor, you need to understand the platform you're building on. Arduino is much more than just a board — it's an entire ecosystem.

The Big Picture

Arduino is an open-source electronics platform combining a hardware board (microcontroller) and a software environment (IDE). Created in 2005 in Italy, it was designed to make electronics accessible to artists, designers, and beginners — not just engineers.

Instead of programming chips in assembly, Arduino gives you simplified C++ where you can blink LEDs, read sensors, and control motors with just a few lines of code.

Key Components of Every Arduino Board

  • Microcontroller (MCU) — the brain. Runs your code. Contains CPU, RAM, Flash memory, and peripherals all in one chip.
  • Digital I/O pins — read or output HIGH/LOW. Connect LEDs, buttons, sensors, motors.
  • Analog input pins — read analog voltages (0–5V) and convert to 0–1023 using built-in ADC.
  • PWM pins — special pins that simulate analog output (dimming LEDs, servo motors).
  • Power pins — 5V, 3.3V, GND to power your sensors.
  • USB interface — connects to your computer for programming and serial communication.
  • Crystal oscillator — sets the clock speed (16 MHz on Uno).
  • Reset button — restarts the currently running program.

How Your Code Runs

  • setup() — runs once when the board powers on. Configure pins, start Serial, initialize sensors.
  • loop() — runs continuously after setup(), forever. This is where you read sensors and react to them.
Arduino C++
void setup() { Serial.begin(9600); // Start serial at 9600 baud pinMode(13, OUTPUT); // Set pin 13 as output } void loop() { digitalWrite(13, HIGH); // LED on delay(1000); digitalWrite(13, LOW); // LED off delay(1000); }

Arduino Uno — Full Guide

The Arduino Uno is the most popular board in the world for learning electronics. It's the standard reference board — when someone says "Arduino," they almost always mean the Uno.

Technical Specifications

SpecificationValue
MicrocontrollerATmega328P
Operating Voltage5V
Input Voltage (recommended)7–12V
Digital I/O Pins14 (6 with PWM)
Analog Input Pins6 (A0–A5)
Flash Memory32 KB
SRAM2 KB
EEPROM1 KB
Clock Speed16 MHz
Board Dimensions68.6mm × 53.4mm

Pin Layout Explained

  • Digital pins 0–13 — general purpose. Pins 0 (RX) and 1 (TX) are for serial. Pin 13 has a built-in LED.
  • PWM pins (~3, ~5, ~6, ~9, ~10, ~11) — use analogWrite(pin, 0-255) for LED dimming and motor speed.
  • Analog pins A0–A5 — read 0–5V. Also usable as digital pins 14–19.
  • Power pins — 3.3V (50mA max), 5V, two GND, Vin (input voltage).
  • I2C pins — A4 = SDA, A5 = SCL.
  • SPI pins — 10 (SS), 11 (MOSI), 12 (MISO), 13 (SCK).
⚠️ Important Limits
Never draw more than 40mA from a single pin or more than 200mA total from all I/O pins combined. Power hungry components directly from the 5V power pin, not from I/O pins.

Arduino Nano — Full Guide

The Nano is essentially a miniaturized Uno. Same brain, much smaller body — perfect when space is tight or you're embedding a controller inside a finished project.

Technical Specifications

SpecificationValue
MicrocontrollerATmega328P (same as Uno)
Operating Voltage5V
Analog Input Pins8 (A0–A7) — 2 more than Uno!
Flash Memory32 KB
SRAM2 KB
Clock Speed16 MHz
Board Dimensions45mm × 18mm — much smaller!
Breadboard friendlyYes — fits directly on a breadboard

What's Different from the Uno?

  • Size — dramatically smaller (45×18mm vs 68×53mm). Perfect for wearables and embedded projects.
  • Breadboard-friendly — its DIP-style layout slots directly into a standard breadboard. The Uno does NOT fit on a breadboard.
  • 2 extra analog pins — A6 and A7 (analog input only — cannot be used as digital I/O).
  • No barrel jack — powered via USB or the Vin pin.
  • Same capabilities otherwise — same MCU, clock speed, memory, and logic.
📌 CH340 Driver Note
Many Nano clones use the CH340G USB chip. Your computer may not recognize it automatically. If your Nano isn't detected, download and install the CH340 driver for your OS. Also set: Tools → Processor → ATmega328P (Old Bootloader).

Arduino Mega — Full Guide

The Mega is the powerhouse of the Arduino family. When your project outgrows the Uno — needing more pins, more memory, or more serial ports — the Mega is the answer.

Technical Specifications

SpecificationValue
MicrocontrollerATmega2560
Digital I/O Pins54 (15 with PWM)
Analog Input Pins16 (A0–A15)
Flash Memory256 KB (8× the Uno!)
SRAM8 KB
Hardware Serial ports4 (Serial, Serial1, Serial2, Serial3)
I2CPins 20 (SDA), 21 (SCL)
Board Dimensions101.5mm × 53.4mm

The 4 Hardware Serial Ports — The Mega's Killer Feature

On the Uno, there's only one UART, usually reserved for debugging. The Mega gives you four independent UART ports running simultaneously:

  • Serial — pins 0/1 — USB/debugging
  • Serial1 — pins 19/18 — e.g., GPS module
  • Serial2 — pins 17/16 — e.g., Bluetooth module
  • Serial3 — pins 15/14 — e.g., fingerprint sensor

When to Choose the Mega

  • Building complex multi-sensor systems (5+ sensors)
  • Using large libraries that consume lots of Flash memory
  • Projects needing many PWM outputs (robots with multiple servos)
  • Connecting multiple serial devices simultaneously
  • 3D printers, CNC machines (RAMPS shield is designed for the Mega)

Decision Guide

Arduino IDE Setup

The Arduino IDE is where you write, compile, and upload your code. Setting it up correctly is the first hands-on step in your sensor journey.

Installing the Arduino IDE

  1. Go to arduino.cc/en/software and download Arduino IDE 2.x
  2. Run the installer for your OS (Windows, macOS, Linux)
  3. Accept all driver installations when prompted
  4. Open Arduino IDE after installation

Configuring for Your Board

  1. Connect your Arduino to the computer via USB
  2. Go to Tools → Board → Arduino AVR Boards
  3. Select your board: Uno, Nano, or Mega
  4. Go to Tools → Port and select your COM port
  5. For Nano clones with CH340: Tools → Processor → ATmega328P (Old Bootloader)

Installing Sensor Libraries

  1. Go to Sketch → Include Library → Manage Libraries
  2. Search for the library (e.g., "DHT sensor library")
  3. Click Install — always click Install All for dependencies

Essential libraries for this course:

  • DHT sensor library by Adafruit — for DHT11 and DHT22
  • Adafruit Unified Sensor — required by Adafruit libraries
  • NewPing — for HC-SR04 ultrasonic sensor
  • OneWire + DallasTemperature — for DS18B20
  • IRremote — for IR receiver modules
✅ The Serial Monitor
Open with Tools → Serial Monitor or Ctrl+Shift+M. Set baud rate to match your Serial.begin() call (usually 9600). Everything you Serial.print() appears here in real time — your best sensor debugging tool.

Pins, Voltage & Power

Understanding how to safely power sensors and wire them correctly is critical. This lesson prevents you from burning out your components.

The Power Pins

PinVoltageMax CurrentUse For
5V5V regulated~500mA (USB)Most 5V sensors
3.3V3.3V regulated~50mA (Uno)3.3V sensors
GND0V reference—Ground for all components
VinInput (7–12V)—External power input

The Golden Rule of Wiring Sensors

Every sensor needs three connections at minimum:

  1. Power (VCC) — connect to 5V (or 3.3V if required)
  2. Ground (GND) — connect to GND. Never skip this.
  3. Data/Signal — connect to the appropriate Arduino pin

Pull-up & Pull-down Resistors

  • Pull-up resistor (10kΩ to 5V) — holds the pin HIGH by default. Activate with pinMode(pin, INPUT_PULLUP).
  • Pull-down resistor (10kΩ to GND) — holds the pin LOW by default.
Arduino C++
// Using internal pull-up resistor void setup() { Serial.begin(9600); pinMode(2, INPUT_PULLUP); // Pin held HIGH; LOW when button pressed } void loop() { if (digitalRead(2) == LOW) Serial.println("Button pressed!"); delay(50); }
⚠️ Safety Rules
1. Never connect 5V directly to an Arduino output pin — use a resistor for LEDs.
2. Never exceed 40mA on a single I/O pin.
3. Always double-check VCC/GND before powering on — reversed polarity destroys sensors.
