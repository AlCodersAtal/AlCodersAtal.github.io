Module 08 · Lesson 1

Project Overview & Hardware

You've now learned every major sensor category in this course. This final module brings everything together — PIR, ultrasonic, gas, and temperature sensors all working as one unified system.

What You're Building

A Multi-Sensor Smart Security & Safety System — a real-world project that combines five sensors and multiple outputs into a working alarm system. Here's what it does:

  • Arm/Disarm toggle — push button switches between ARMED and DISARMED states. Green LED = disarmed. Red LED = armed.
  • Motion intrusion detection — when armed, both PIR and HC-SR04 are monitored. Motion detected OR object closer than 40cm triggers the intrusion alarm.
  • Gas / smoke alarm — MQ-2 is always active, even when disarmed. If gas level exceeds threshold, three rapid beeps sound immediately.
  • Temperature alert — DHT22 always monitors temperature. If above 40°C (fire indicator), a continuous alarm triggers regardless of arm state.
  • 10-second status report — every 10 seconds, a full system snapshot is printed to the Serial Monitor: arm state, PIR, distance, gas level, temperature, humidity.

Components Required

  • 1× Arduino Uno (or Nano / Mega)
  • 1× HC-SR501 PIR motion sensor
  • 1× HC-SR04 ultrasonic distance sensor
  • 1× MQ-2 gas / smoke sensor module
  • 1× DHT22 temperature & humidity sensor
  • 1× Momentary push button
  • 1× Green LED + 220Ω resistor
  • 1× Red LED + 220Ω resistor
  • 1× Passive buzzer
  • 1× Breadboard + assorted jumper wires
  • 1× USB cable to connect Arduino to computer

Libraries Needed

  • DHT sensor library by Adafruit — for DHT22
  • Adafruit Unified Sensor — required dependency

Install: Sketch → Include Library → Manage Libraries → search "DHT sensor library" → click Install All.

System State Logic Map

  • Startup (0–60s) — all sensors initialize. Serial Monitor shows countdown. Green LED blinks slowly.
  • DISARMED — green LED solid on. Intrusion detection inactive. Gas + temperature alarms still running.
  • ARMED — red LED solid on. All alarms active. PIR HIGH or distance < 40cm = intrusion alarm (rapid buzz + red LED flash).
  • Gas alarm (any state) — three short beeps, "⚠ GAS DETECTED" printed.
  • Temp alarm (any state) — continuous buzzer, "🔥 HIGH TEMP" printed.
  • Button press — debounced toggle: ARMED ↔ DISARMED with confirmation beep.
✅ Wiring Tip — Power Rail
On your breadboard, connect the 5V pin to the red (+) power rail and GND to the blue (−) rail. Then all components can share power cleanly without running wires back to Arduino each time.
⚠️ MQ-2 Current Draw
The MQ-2 heater draws around 150–200mA. Do not power it through an Arduino I/O pin — always connect VCC directly to the 5V power pin. Powering via USB is fine for testing; for standalone use, use a 9V adapter through Vin.

Writing the Code — Part 1

We build the system in layers. Part 1 covers the setup, pin definitions, helper functions, and the arm/disarm toggle.

Step 1 — Includes, Pins & Global Variables

All pin numbers and configuration constants go at the top. This makes tuning easy — change a value in one place, it updates everywhere.

Arduino C++ — Includes, Pins & Globals
#include "DHT.h" // ── PIN DEFINITIONS ─────────────────────── #define PIR_PIN 7 #define TRIG_PIN 9 #define ECHO_PIN 10 #define MQ2_PIN A0 #define DHT_PIN 2 #define BTN_PIN 4 #define GREEN_LED 6 #define RED_LED 5 #define BUZZER 8 // ── THRESHOLDS ───────────────────────────── #define GAS_THRESHOLD 600 // MQ-2 raw value — tune after preheat #define TEMP_THRESHOLD 40.0 // °C — triggers temp alarm #define DIST_THRESHOLD 40 // cm — intruder too close // ── SENSOR OBJECT ────────────────────────── DHT dht(DHT_PIN, DHT22); // ── STATE ────────────────────────────────── bool systemArmed = false; int lastBtnState = HIGH; unsigned long lastDebounce = 0; unsigned long lastStatus = 0; unsigned long lastMotionMsg = 0;

Step 2 — setup()

Configure all pins, start Serial and DHT, then wait for the PIR's 60-second warm-up — blinking the green LED so the user knows the system is alive.

Arduino C++ — setup()
void setup() { Serial.begin(9600); dht.begin(); pinMode(PIR_PIN, INPUT); pinMode(TRIG_PIN, OUTPUT); pinMode(ECHO_PIN, INPUT); pinMode(BTN_PIN, INPUT_PULLUP); pinMode(GREEN_LED, OUTPUT); pinMode(RED_LED, OUTPUT); pinMode(BUZZER, OUTPUT); Serial.println("=== AlCoders Security System ==="); Serial.println("PIR warming up — please wait 60s..."); for (int i = 60; i > 0; i--) { Serial.print("Ready in: "); Serial.print(i); Serial.println("s"); digitalWrite(GREEN_LED, HIGH); delay(500); digitalWrite(GREEN_LED, LOW); delay(500); } digitalWrite(GREEN_LED, HIGH); // default state: disarmed Serial.println("✅ System ready! Press button to ARM."); }

Step 3 — Arm/Disarm Toggle

The button is read with INPUT_PULLUP (LOW = pressed). We debounce with millis() and toggle state on each press.

Arduino C++ — checkButton()
void checkButton() { int reading = digitalRead(BTN_PIN); if (reading != lastBtnState) lastDebounce = millis(); if ((millis() - lastDebounce) > 50) { if (reading == LOW && lastBtnState == HIGH) { systemArmed = !systemArmed; if (systemArmed) { digitalWrite(GREEN_LED, LOW); digitalWrite(RED_LED, HIGH); Serial.println("🔴 SYSTEM ARMED"); tone(BUZZER, 1000, 200); // short confirm beep } else { digitalWrite(RED_LED, LOW); digitalWrite(GREEN_LED, HIGH); Serial.println("🟢 SYSTEM DISARMED"); noTone(BUZZER); } } } lastBtnState = reading; }

Writing the Code — Part 2

Part 2 covers the sensor helper functions, the three alarm routines, and the status reporter — each written as its own clean function.

Helper — Read Ultrasonic Distance

Arduino C++ — getDistance()
long getDistance() { digitalWrite(TRIG_PIN, LOW); delayMicroseconds(2); digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10); digitalWrite(TRIG_PIN, LOW); long dur = pulseIn(ECHO_PIN, HIGH, 30000); // 30ms timeout if (dur == 0) return 999; // out of range — treat as safe return dur / 58; }

Alarm — Intrusion (when armed)

Arduino C++ — intrusionAlarm()
void intrusionAlarm(const char* source) { if (millis() - lastMotionMsg < 3000) return; // throttle prints lastMotionMsg = millis(); Serial.print("🚨 INTRUDER DETECTED via "); Serial.println(source); for (int i = 0; i < 5; i++) { digitalWrite(RED_LED, LOW); tone(BUZZER, 2400, 150); delay(200); digitalWrite(RED_LED, HIGH); delay(150); } }

Alarm — Gas Detection (always active)

Arduino C++ — checkGas()
void checkGas() { int gasLevel = analogRead(MQ2_PIN); if (gasLevel > GAS_THRESHOLD) { Serial.print("⚠ GAS DETECTED! Level: "); Serial.println(gasLevel); for (int i = 0; i < 3; i++) { tone(BUZZER, 1800, 200); delay(300); } delay(2000); // wait before re-checking } }

Alarm — Temperature Alert (always active)

Arduino C++ — checkTemperature()
void checkTemperature() { float tempC = dht.readTemperature(); if (isnan(tempC)) return; if (tempC > TEMP_THRESHOLD) { Serial.print("🔥 HIGH TEMP ALERT: "); Serial.print(tempC); Serial.println("°C"); tone(BUZZER, 3000); // continuous tone until temp drops delay(1000); noTone(BUZZER); } }

Status Reporter — Every 10 Seconds

Arduino C++ — printStatus()
void printStatus() { if (millis() - lastStatus < 10000) return; lastStatus = millis(); float tempC = dht.readTemperature(); float humidity = dht.readHumidity(); long dist = getDistance(); int gas = analogRead(MQ2_PIN); bool pir = (digitalRead(PIR_PIN) == HIGH); Serial.println("──────────── STATUS ────────────"); Serial.print("System : "); Serial.println(systemArmed ? "ARMED 🔴" : "DISARMED 🟢"); Serial.print("PIR : "); Serial.println(pir ? "Motion" : "Clear"); Serial.print("Distance: "); Serial.print(dist); Serial.println(" cm"); Serial.print("Gas MQ-2: "); Serial.println(gas); if (!isnan(tempC)) { Serial.print("Temp : "); Serial.print(tempC); Serial.println("°C"); Serial.print("Humidity: "); Serial.print(humidity); Serial.println("%"); } Serial.println("────────────────────────────────"); }

Testing & Calibration

A project isn't done when the code compiles — it's done when it behaves reliably. Follow this step-by-step testing procedure.

Step-by-Step Test Procedure

  1. Upload the sketch. Open Serial Monitor at 9600 baud. You should see the 60-second countdown with green LED blinking.
  2. After warmup — Serial prints "Ready". Green LED stays solid. System is DISARMED.
  3. Test button toggle — press the button. Serial should print "SYSTEM ARMED" and the red LED turns on. Press again: "SYSTEM DISARMED", green LED returns.
  4. Test PIR (armed) — arm the system. Walk in front of the PIR sensor. Alarm should sound and "INTRUDER via PIR" should print.
  5. Test ultrasonic (armed) — arm the system. Hold your hand 20cm in front of the HC-SR04. Alarm sounds and "INTRUDER via Ultrasonic" prints.
  6. Test gas alarm (any state) — disarm the system first. Bring a lit match near the MQ-2 briefly (careful!). "GAS DETECTED" should print with three beeps.
  7. Test status report — wait 10 seconds and verify the status block prints with sensible values.
  8. Test disarmed safety — with the system DISARMED, trigger the gas sensor. Alarm should still sound — gas alarm is always active.

Calibrating Your Thresholds

  • GAS_THRESHOLD — Open Serial Monitor. Read the "Gas" line in the status report in clean air. Your threshold should be about 100–150 points above the clean-air reading.
  • DIST_THRESHOLD — adjust based on the size of the area you want to protect. 40cm is good for a desk or doorway sensor.
  • TEMP_THRESHOLD — 40°C is suitable for indoor rooms. Lower to 35°C for sensitive areas.

Common Problems & Fixes

  • PIR triggers constantly — it hasn't warmed up yet, or is positioned near a heat vent / window. Reposition and wait the full 60 seconds.
  • DHT returns NaN — wait 2 seconds between reads (already handled), check data pin wiring, ensure module has pull-up resistor.
  • MQ-2 always shows high values — it needs a 20-minute preheat at first use. Readings will settle. Recalibrate threshold after preheat.
  • Button not toggling — check that one leg goes to pin 4 and the other to GND, and that INPUT_PULLUP is set.
  • HC-SR04 always reads 999cm — check 5V connection and that Trig/Echo are not swapped.
✅ Extensions to Try
Once your base project works, try adding: an LCD display showing arm state and live readings, an RGB LED that colour-codes the alarm type, a buzzer tone() melody for the arm/disarm sound, or logging readings to SD card with a timestamp.
