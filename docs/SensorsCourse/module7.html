Soil Moisture — Principle & Types Analog + Digital

Soil moisture sensors are the foundation of smart irrigation systems. Understanding both types helps you choose the right one for your project's lifespan requirements.

Why Measure Soil Moisture?

Plants need a specific range of soil moisture — too dry and they wilt, too wet and roots rot. Traditional schedules waste water by ignoring actual soil conditions. A soil moisture sensor lets your Arduino water plants only when needed, saving water and keeping plants healthier.

Type 1: Resistive Soil Moisture Sensor (YL-69 / FC-28)

The most common cheap sensor (the two-probe "fork" type). Works by passing a small current between two exposed metal probes and measuring resistance. Dry soil has very high resistance. Wet soil conducts electricity well and has low resistance.

  • Pros: Very cheap (~$1), simple, no calibration needed for basic use
  • Cons: Exposed metal probes corrode (rust) over time due to electrolysis. Accuracy degrades after months of continuous use in soil. Not recommended for permanent installations.

Type 2: Capacitive Soil Moisture Sensor

A better design with no exposed probes. Measures the dielectric permittivity of the soil — water has a much higher dielectric constant than dry soil (~80 vs ~4). The sensor's PCB traces form a capacitor; as soil moisture changes, the capacitance changes and the onboard circuit converts this to a voltage.

  • Pros: No corrosion (fully coated PCB), much longer lifespan, stable readings
  • Cons: Slightly more expensive (~$3–5). Needs calibration for accurate % readings.

⚠️ Don't Power Resistive Sensors Continuously
Continuously passing current through the probes accelerates corrosion. Best practice: power the sensor from a digital output pin, pull it HIGH just before reading, then LOW immediately after. This significantly extends sensor life.

Soil Moisture Sensor with Arduino

Let's build a complete smart plant watering system that monitors soil moisture and automatically controls a water pump.

Arduino C++ — Smart Plant Watering System
const int SENSOR_AO = A0; const int SENSOR_VCC = 6; // power sensor only when reading const int PUMP_PIN = 8; const int DRY_THRESH = 700; // above = dry — tune to your soil const int WET_THRESH = 300; // below = wet enough int readSoilMoisture() { digitalWrite(SENSOR_VCC, HIGH); delay(100); int val = analogRead(SENSOR_AO); digitalWrite(SENSOR_VCC, LOW); return val; } void setup() { Serial.begin(9600); pinMode(SENSOR_VCC, OUTPUT); pinMode(PUMP_PIN, OUTPUT); digitalWrite(PUMP_PIN, LOW); } void loop() { int moisture = readSoilMoisture(); int percent = map(moisture, 1023, 0, 0, 100); // invert Serial.print("Soil: "); Serial.print(percent); Serial.print("%"); if (moisture > DRY_THRESH) { Serial.println(" — DRY 💧 Pumping water..."); digitalWrite(PUMP_PIN, HIGH); delay(3000); digitalWrite(PUMP_PIN, LOW); delay(30000); // wait for water to absorb } else if (moisture < WET_THRESH) { Serial.println(" — WET ✅ No watering needed"); digitalWrite(PUMP_PIN, LOW); } else { Serial.println(" — OK 🌱 Moisture is fine"); digitalWrite(PUMP_PIN, LOW); } delay(10000); }
✅ Calibrating Your Thresholds
1. Insert sensor into completely dry soil — note the raw value (typically 700–900).
2. Water the soil thoroughly — note the wet value (typically 200–400).
3. Set DRY_THRESH slightly below your dry reading, WET_THRESH slightly above your wet reading.

Gas Sensors — MQ Series Analog + Digital

The MQ series are the most popular gas sensors for Arduino — same housing, same interface, but each model detects a different gas or gas mixture.

How MQ Sensors Work — Metal Oxide Semiconductor

MQ sensors use a tin dioxide (SnO₂) layer on a ceramic substrate, heated by a nichrome wire coil to ~200–300°C (this is why MQ sensors get warm during operation):

  1. The heater raises the SnO₂ element to operating temperature
  2. Oxygen from air adsorbs onto the SnO₂ surface, trapping electrons and increasing resistance
  3. When target gas molecules contact the surface, they react with the adsorbed oxygen, releasing electrons and decreasing resistance
  4. This resistance drop is converted to a voltage change by a load resistor, which Arduino reads via the analog output

⚠️ 20-Minute Preheat Required
MQ sensors need the heater to warm the metal oxide before readings are accurate. Preheat for at least 20 minutes at first use, and 2–3 minutes on subsequent power-ons. First readings after power-on will always be high/unstable — this is normal.

MQ Sensors with Arduino — Full Code

Let's wire up the MQ-2 and build a complete three-level gas safety alarm with LED indicators.

Arduino C++ — MQ-2 Gas & Smoke Alarm
const int MQ2_AO = A0; const int MQ2_DO = 7; const int GREEN_LED = 3; const int YELLOW_LED = 5; const int RED_LED = 6; const int BUZZER = 8; const int WARN_LEVEL = 400; const int ALARM_LEVEL = 700; void setup() { Serial.begin(9600); pinMode(MQ2_DO, INPUT); pinMode(GREEN_LED, OUTPUT); pinMode(YELLOW_LED, OUTPUT); pinMode(RED_LED, OUTPUT); pinMode(BUZZER, OUTPUT); Serial.println("MQ-2 Preheating (60s)..."); delay(60000); Serial.println("✅ Ready"); } void loop() { int gasLevel = analogRead(MQ2_AO); Serial.print("Gas: "); Serial.print(gasLevel); // Clear all indicators digitalWrite(GREEN_LED, LOW); digitalWrite(YELLOW_LED, LOW); digitalWrite(RED_LED, LOW); digitalWrite(BUZZER, LOW); if (gasLevel >= ALARM_LEVEL) { digitalWrite(RED_LED, HIGH); digitalWrite(BUZZER, HIGH); Serial.println(" ❌ DANGER!"); } else if (gasLevel >= WARN_LEVEL) { digitalWrite(YELLOW_LED, HIGH); Serial.println(" ⚠ WARNING"); } else { digitalWrite(GREEN_LED, HIGH); Serial.println(" ✅ CLEAN"); } delay(500); }

Project: Smart Plant Monitor

Combine soil moisture, temperature, humidity, and air quality sensors into a complete plant health reporting system.

🔨 Build: Smart Plant Health Monitor

Read soil moisture, DHT22, and MQ-135 together to give a complete plant environment health report.

  1. Wire: DHT22 (pin 2), soil moisture sensor (A0, VCC on pin 6), MQ-135 AO (A1), buzzer (pin 8)
  2. Every 5 seconds, read all three sensors
  3. Score each: temperature OK/HIGH/LOW, humidity OK/DRY/HUMID, soil OK/DRY/WATERLOGGED, air OK/POOR
  4. Print a "Plant Health Report" to Serial Monitor with all scores
  5. Calculate overall health: GREAT (all OK) / OKAY (1 issue) / NEEDS ATTENTION (2+ issues)
  6. Trigger buzzer with 1 beep for OKAY, 3 beeps for NEEDS ATTENTION
