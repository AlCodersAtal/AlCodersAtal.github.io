DHT11 vs DHT22 — Full Comparison

The DHT11 and DHT22 are the most popular temperature and humidity sensors for Arduino. They look almost identical but have important differences.

SpecificationDHT11 (Blue)DHT22 / AM2302 (White)
Temperature range0°C to 50°C-40°C to 80°C
Temp accuracy±2°C±0.5°C
Temp resolution1°C (integer only)0.1°C (decimal)
Humidity range20% – 80% RH0% – 100% RH
Humidity accuracy±5% RH±2–5% RH
Humidity resolution1% RH0.1% RH
Sampling rateOnce per secondOnce per 2 seconds
Operating voltage3.3V – 5.5V3.3V – 5.5V
Price~$1–2~$3–5

Which Should You Use?

  • Use DHT11 for: learning projects, simple on/off decisions, budget builds, room-temperature-only applications
  • Use DHT22 for: anything requiring accuracy, outdoor use, data logging, greenhouses, incubators
  • The DHT22 is almost always worth the extra ~$2 for any serious project
📌 DHT22 = AM2302
The DHT22 and AM2302 are the same sensor. AM2302 just refers to the version with a built-in pull-up resistor and wiring pigtail. They use identical code.

How DHT Sensors Work Inside

Understanding the DHT's internal working explains why you can only read it once every 1–2 seconds and helps you troubleshoot common issues.

What's Inside the DHT Sensor

  • Capacitive humidity sensor — a polymer film between two electrodes. As humidity increases, the film absorbs water vapor and its capacitance changes. The internal MCU measures this and converts it to % RH.
  • NTC thermistor — a resistor whose resistance decreases as temperature increases. The internal MCU reads it and converts to °C.
  • Internal 8-bit MCU — handles measurement, conversion, calibration, and serial communication. This is why DHT sensors are easy to use — all the hard work happens inside the sensor.

The Single-Wire Communication Protocol

  1. Arduino pulls DATA line LOW for at least 18ms — the "wake up" signal
  2. Arduino releases DATA line HIGH
  3. Sensor responds: LOW 80µs then HIGH 80µs — handshake
  4. Sensor sends 40 bits: 16 bits humidity + 16 bits temperature + 8-bit checksum
  5. Each bit: ~28µs HIGH = bit 0, ~70µs HIGH = bit 1
  6. The library decodes all this and returns a clean float value

This entire process takes time — which is why you must wait 1–2 seconds between readings. Reading too fast returns NaN (Not a Number).

Why Readings Sometimes Fail

  • Too fast reading — faster than once per 2 seconds (DHT22) or 1 second (DHT11)
  • Missing pull-up resistor — the data line needs a 10kΩ pull-up to 5V. Modules usually have it built in; bare sensors don't.
  • Long data wire — cables over ~20cm cause signal integrity issues
  • Power supply noise — add a 100nF ceramic capacitor between VCC and GND to stabilize readings

DHT Sensor with Arduino — Complete Code

Let's wire up the DHT sensor and build a complete temperature/humidity monitor with heat index calculation.

Wiring DHT11 / DHT22

DHT PinConnect ToNotes
Pin 1 (VCC)5VLeft pin when facing sensor
Pin 2 (DATA)Digital Pin 2 + 10kΩ to 5VPull-up required for bare sensor
Pin 3Not connected
Pin 4 (GND)GNDRight pin when facing sensor

Module (3-pin version): VCC → 5V, DATA → Pin 2, GND → GND. No external pull-up needed.

Arduino C++ — DHT with Heat Index
#include "DHT.h" #define DHT_PIN 2 #define DHT_TYPE DHT22 // change to DHT11 for DHT11 DHT dht(DHT_PIN, DHT_TYPE); void setup() { Serial.begin(9600); dht.begin(); Serial.println("=== DHT Monitor ==="); } void loop() { delay(2000); // MUST wait 2s between DHT22 reads float humidity = dht.readHumidity(); float tempC = dht.readTemperature(); float tempF = dht.readTemperature(true); if (isnan(humidity) || isnan(tempC)) { Serial.println("❌ Failed to read from DHT!"); return; } float heatIndex = dht.computeHeatIndex(tempC, humidity, false); Serial.println("----------------------------"); Serial.print("Humidity: "); Serial.print(humidity); Serial.println("%"); Serial.print("Temperature: "); Serial.print(tempC); Serial.println("°C"); Serial.print(" : "); Serial.print(tempF); Serial.println("°F"); Serial.print("Feels like: "); Serial.print(heatIndex); Serial.println("°C"); if (humidity > 70) Serial.println("⚠ High humidity — muggy"); if (tempC > 35) Serial.println("🔥 High temperature!"); if (tempC < 10) Serial.println("🥶 Low temperature!"); }
✅ Heat Index Explained
The heat index ("feels like" temperature) combines temperature and humidity to give a human-perceived temperature. At 35°C with 80% humidity, it feels like 42°C. The DHT library's computeHeatIndex() uses the Steadman formula to calculate this.

DS18B20 Waterproof Temperature Sensor 1-Wire

The DS18B20 can go underwater, into soil, or into liquids. Multiple sensors can share a single data pin using the 1-Wire protocol.

What Makes the DS18B20 Special

  • Waterproof probe — sealed stainless steel, safe in water, soil, food, chemicals
  • 1-Wire protocol — only one data wire plus GND. Multiple DS18B20 sensors can share the same data wire — each has a unique 64-bit address
  • 12-bit resolution — 0.0625°C precision
  • Range: -55°C to +125°C — handles boiling water and deep freezes

Arduino C++ — DS18B20 Temperature Sensor
#include <OneWire.h> #include <DallasTemperature.h> OneWire oneWire(4); DallasTemperature sensors(&oneWire); void setup() { Serial.begin(9600); sensors.begin(); Serial.print("Found "); Serial.print(sensors.getDeviceCount()); Serial.println(" DS18B20 sensor(s)"); } void loop() { sensors.requestTemperatures(); float tempC = sensors.getTempCByIndex(0); if (tempC == DEVICE_DISCONNECTED_C) { Serial.println("Sensor disconnected!"); } else { Serial.print("Temperature: "); Serial.print(tempC, 2); Serial.println("°C"); } delay(1000); }

Project: Mini Weather Station

Build a weather station that displays temperature, humidity, heat index, and a comfort level message on an LCD.

🔨 Build: Mini Weather Station with LCD

Combine DHT22 and a 16x2 I2C LCD display to build a real weather station with comfort level feedback.

  1. Wire DHT22 (pin 2) and a 16x2 LCD display via I2C (VCC, GND, SDA=A4, SCL=A5)
  2. Display temperature on LCD row 1: "Temp: 27.3°C"
  3. Display humidity on LCD row 2: "Hum: 65% MUGGY"
  4. Comfort labels: COLD / COOL / OK / WARM / HOT for temperature; DRY / OK / HUMID for humidity
  5. Send full readings including heat index to Serial Monitor every 2 seconds
  6. Bonus: add an RGB LED that glows blue when cold, green when comfortable, red when hot
