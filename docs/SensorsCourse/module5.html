PIR Motion Sensor — Deep Dive Digital

PIR sensors are the technology behind automatic lights, burglar alarms, and smart home motion detection. Let's understand exactly how they detect human presence.

The Physics Behind PIR

All objects above absolute zero emit infrared radiation. The human body at ~37°C emits IR most strongly in the 9–10 micrometer wavelength range. The PIR sensor contains a pyroelectric crystal — a material that generates a small electric charge when its temperature changes. The sensor doesn't measure absolute temperature; it detects changes in IR radiation. When a warm body moves across its field of view, the IR level shifts rapidly and the crystal generates a voltage pulse.

The Fresnel Lens

The distinctive white dome on a PIR sensor is a Fresnel lens — a segmented optical lens that focuses and divides the sensor's field of view into multiple zones. As a person walks past, they alternately enter and leave these zones, creating rapid IR changes that the sensor detects as motion. Without the lens, the PIR would barely detect anything.

HC-SR501 PIR Module — Adjustable Parameters

  • Sensitivity potentiometer — detection range from ~3m (CCW) to ~7m (CW)
  • Time delay potentiometer — output stays HIGH for ~5 seconds to ~5 minutes after detection
  • Trigger mode jumper:
    • Single trigger (H) — output goes HIGH for set time, then LOW. Waits for timer before retriggering.
    • Repeat trigger (L) — output stays HIGH while motion is continuously detected. Resets timer each time.

⚠️ 60-Second Initialization
The HC-SR501 requires ~60 seconds to stabilize after power-on. During this time it will trigger false positives. Always add a 60-second startup delay in your code before reading the sensor.

Arduino C++ — PIR Motion Alarm with Warmup
const int PIR_PIN = 7; const int LED_PIN = 13; const int BUZZ_PIN = 8; unsigned long motionTime = 0; void setup() { Serial.begin(9600); pinMode(PIR_PIN, INPUT); pinMode(LED_PIN, OUTPUT); pinMode(BUZZ_PIN, OUTPUT); Serial.println("PIR warming up..."); for (int i = 60; i > 0; i--) { Serial.print("Ready in: "); Serial.print(i); Serial.println("s"); delay(1000); } Serial.println("✅ PIR Ready!"); } void loop() { if (digitalRead(PIR_PIN) == HIGH) { digitalWrite(LED_PIN, HIGH); digitalWrite(BUZZ_PIN, HIGH); if (millis() - motionTime > 2000) { Serial.println("🚨 MOTION DETECTED!"); motionTime = millis(); } } else { digitalWrite(LED_PIN, LOW); digitalWrite(BUZZ_PIN, LOW); } delay(50); }
📌 millis() vs delay()
We use millis() for timing instead of delay(). The delay() function blocks everything — while delaying, Arduino can't read other sensors. millis() checks "has enough time passed?" without freezing the program. Always prefer millis() in real projects.

Ultrasonic HC-SR04 — How It Works Digital/Timing

The HC-SR04 lets your Arduino "see" how far objects are, just like a bat using echolocation or a submarine using SONAR.

The Physics of Ultrasound

Ultrasound is sound above 20,000 Hz — above human hearing range. The HC-SR04 operates at 40kHz. Sound travels through air at approximately 343 m/s at 20°C. The sensor uses this known speed to calculate distance by measuring the travel time of a sound pulse.

How Distance Is Measured — Step by Step

  1. Arduino sends a 10µs HIGH pulse to the Trigger pin
  2. The sensor emits 8 ultrasonic pulses at 40kHz from the emitter (T)
  3. Pulses travel outward, hit an object, and reflect back
  4. The receiver (R) detects the returning pulses
  5. The Echo pin goes HIGH for exactly the duration the sound was traveling
  6. You measure the Echo duration using pulseIn()
  7. Distance = (Echo duration × speed of sound) ÷ 2 (divide by 2: sound travels there AND back)

The Distance Formula

If Echo duration = T microseconds:

Distance (cm) = T ÷ 58.2

Why 58.2? Sound travels 343 m/s = 0.0343 cm/µs. There and back (÷2) = 0.01715 cm/µs. Inverted: 1/0.01715 ≈ 58.2 µs per cm.

HC-SR04 with Arduino — Full Implementation

Let's wire up the ultrasonic sensor and build a complete distance measurement system with proximity alarm.

Arduino C++ — HC-SR04 Distance Measurement
const int TRIG_PIN = 9; const int ECHO_PIN = 10; const int BUZZ_PIN = 8; const int ALERT_CM = 20; long readDistance() { digitalWrite(TRIG_PIN, LOW); delayMicroseconds(2); digitalWrite(TRIG_PIN, HIGH); delayMicroseconds(10); digitalWrite(TRIG_PIN, LOW); long duration = pulseIn(ECHO_PIN, HIGH, 30000); if (duration == 0) return -1; // out of range return duration / 58.2; } void setup() { Serial.begin(9600); pinMode(TRIG_PIN, OUTPUT); pinMode(ECHO_PIN, INPUT); pinMode(BUZZ_PIN, OUTPUT); } void loop() { long dist = readDistance(); if (dist == -1) { Serial.println("Out of range"); } else { Serial.print("Distance: "); Serial.print(dist); Serial.println(" cm"); digitalWrite(BUZZ_PIN, dist < ALERT_CM ? HIGH : LOW); } delay(60); }

Using the NewPing Library (Simpler)

Arduino C++ — HC-SR04 with NewPing Library
#include <NewPing.h> NewPing sonar(9, 10, 400); // trig, echo, max distance cm void setup() { Serial.begin(9600); } void loop() { delay(50); unsigned int dist = sonar.ping_cm(); // 0 = out of range Serial.print("Distance: "); Serial.print(dist); Serial.println(" cm"); }

Project: Obstacle Detector

Build a parking-sensor style system — the buzzer beeps faster as objects get closer, with LED zone indicators.

🔨 Build: Parking Sensor with Zone Indicators

Combine HC-SR04 distance sensing with a buzzer and three LEDs to create a real parking-sensor style proximity alert system.

  1. Wire HC-SR04 (Trig=9, Echo=10), passive buzzer (pin 8), green LED (pin 3), yellow LED (pin 5), red LED (pin 6)
  2. Read distance every 100ms
  3. Over 80cm: buzzer silent, green LED on
  4. 20–80cm: buzzer beeps slowly (500ms interval), yellow LED on
  5. 5–20cm: buzzer beeps fast (100ms interval), red LED on
  6. Under 5cm: continuous solid buzzer tone, red LED flashing
