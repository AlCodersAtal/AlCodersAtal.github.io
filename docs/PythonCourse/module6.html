Lists

A list is an ordered, mutable collection of items. It can hold any data type — even mixed types. Lists are the most-used data structure in Python.

Creating Lists

Python
fruits = ["apple", "banana", "cherry"] numbers = [1, 2, 3, 4, 5] mixed = [1, "hello", 3.14, True] empty = [] # Access by index print(fruits[0]) # apple print(fruits[-1]) # cherry (last) print(len(fruits)) # 3 # Modify a list fruits[1] = "blueberry" print(fruits) # ['apple', 'blueberry', 'cherry']

Slicing

Python
nums = [0, 1, 2, 3, 4, 5] print(nums[1:4]) # [1, 2, 3] print(nums[:3]) # [0, 1, 2] print(nums[3:]) # [3, 4, 5] print(nums[::2]) # [0, 2, 4] — every 2nd item print(nums[::-1]) # [5, 4, 3, 2, 1, 0] — reversed

List Methods & Slicing

Lists come with many built-in methods for adding, removing, sorting, and searching items.

Essential List Methods

Python
items = [3, 1, 4, 1, 5] items.append(9) # Add to end → [3,1,4,1,5,9] items.insert(0, 99) # Insert at index 0 → [99,3,1,4,...] items.remove(1) # Remove first 1 found popped = items.pop() # Remove & return last item popped2 = items.pop(0) # Remove & return item at index 0 items.sort() # Sort in place (ascending) items.sort(reverse=True) # Sort descending items.reverse() # Reverse in place print(items.count(1)) # How many 1s exist print(items.index(4)) # Index of first 4 print(sorted(items)) # Returns new sorted list, original unchanged a = [1, 2] b = [3, 4] a.extend(b) # [1, 2, 3, 4] — add all from b print(a + b) # [1, 2, 3, 4, 3, 4] — concatenate

Checking Membership

Python
fruits = ["apple", "banana", "cherry"] print("apple" in fruits) # True print("mango" not in fruits) # True

Tuples

Tuples are like lists, but immutable — once created, they cannot be changed. Use tuples for data that should never be modified.

Creating Tuples

Python
point = (3, 7) rgb = (255, 128, 0) single = (42,) # Note trailing comma — required for single-item tuple empty = () # Access like list print(point[0]) # 3 print(len(rgb)) # 3 # Unpacking x, y = point print(f"x={x}, y={y}") # Immutable — this would raise TypeError: # point[0] = 10 ← not allowed!

When to Use Tuples vs Lists

TuplesLists
Fixed data (coordinates, RGB)Data that changes (shopping cart)
Dictionary keys (hashable)Cannot be dict keys
Slightly fasterMore methods available
Signals "don't modify this"Signals "this may change"

Dictionaries

Dictionaries store key-value pairs. Look up a value by its key — like a real dictionary where you look up a word to find its meaning.

Creating and Using Dicts

Python
student = { "name": "Alice", "age": 20, "grade": "A" } # Access values print(student["name"]) # Alice print(student.get("age")) # 20 print(student.get("phone", "N/A")) # N/A (default if key missing) # Add / update student["email"] = "alice@example.com" student["age"] = 21 # Delete del student["grade"] student.pop("email") # Iterate for key, value in student.items(): print(f"{key}: {value}") # Check key exists print("name" in student) # True

Dict Methods

Python
d = {"a": 1, "b": 2, "c": 3} print(d.keys()) # dict_keys(['a', 'b', 'c']) print(d.values()) # dict_values([1, 2, 3]) print(d.items()) # dict_items([('a',1),('b',2),('c',3)]) d2 = {"d": 4} d.update(d2) # Merge d2 into d
Module 6 · Lesson 5

Sets

A set is an unordered collection of unique items. Perfect for removing duplicates and testing membership fast.

Creating and Using Sets

Python
colours = {"red", "blue", "green", "blue"} print(colours) # {'red', 'blue', 'green'} — duplicate removed colours.add("yellow") colours.remove("red") print("blue" in colours) # True — O(1) lookup! # Remove duplicates from a list dupes = [1, 2, 2, 3, 3, 3] unique = list(set(dupes)) # [1, 2, 3]

Set Operations (Maths!)

Python
a = {1, 2, 3, 4} b = {3, 4, 5, 6} print(a | b) # {1,2,3,4,5,6} — union print(a & b) # {3, 4} — intersection print(a - b) # {1, 2} — difference print(a ^ b) # {1,2,5,6} — symmetric difference
Module 6 · Lesson 6

List Comprehensions

List comprehensions let you create new lists in a single, readable line. They're one of Python's most elegant features.

Basic Syntax

Python
# [expression for item in iterable] # Old way — loop squares = [] for x in range(1, 6): squares.append(x**2) # List comprehension — same result, one line! squares = [x**2 for x in range(1, 6)] print(squares) # [1, 4, 9, 16, 25] # With a condition evens = [x for x in range(1, 11) if x % 2 == 0] print(evens) # [2, 4, 6, 8, 10] # Transform strings words = ["hello", "world", "python"] upper = [w.upper() for w in words] print(upper) # ['HELLO', 'WORLD', 'PYTHON'] # Dict comprehension sq_dict = {x: x**2 for x in range(1, 6)} print(sq_dict) # {1:1, 2:4, 3:9, 4:16, 5:25}

Nested Data Structures

Real-world data is often complex — lists of dicts, dicts of lists, and deeper nesting. Python handles it naturally.

List of Dicts (Common Pattern)

Python
students = [ {"name": "Alice", "score": 92}, {"name": "Bob", "score": 78}, {"name": "Charlie", "score": 85}, ] # Access print(students[0]["name"]) # Alice # Loop through for s in students: print(f"{s['name']}: {s['score']}") # Sort by score students.sort(key=lambda s: s["score"], reverse=True) # Filter — only students with score >= 85 top = [s for s in students if s["score"] >= 85]

Dict of Lists

Python
school = { "Class A": ["Alice", "Bob"], "Class B": ["Charlie", "Diana"], } school["Class A"].append("Eve") print(school["Class A"]) # ['Alice', 'Bob', 'Eve']
