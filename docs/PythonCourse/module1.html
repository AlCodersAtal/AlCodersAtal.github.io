What is Python?

Before writing a single line of code, let's understand what Python actually is, where it came from, and why it's one of the most loved programming languages in the world.

A Brief History

Python was created by Guido van Rossum and first released in 1991. The name comes from the British comedy show Monty Python's Flying Circus — not the snake. Guido wanted a language that was fun to use, and the name reflected that spirit.

Today Python is maintained by the Python Software Foundation and is on version 3.x. It's completely free and open-source.

What Makes Python Special?

Python is a high-level, interpreted, general-purpose programming language. Let's break that down:

  • High-level — You write code that reads almost like plain English. Python handles low-level details (like memory management) for you.
  • Interpreted — Python runs your code line-by-line at runtime. You don't need to compile your code first (unlike C or Java).
  • General-purpose — Python can be used for almost anything: web apps, data science, automation, AI, games, scripts, and more.

Why Learn Python?

🔰 Easy to Learn

Python syntax is clean and readable. Beginners can write useful programs within hours.

🌍 Hugely Popular

Consistently ranked #1 or #2 in global programming language surveys (TIOBE, Stack Overflow).

💼 High Demand

Python developers are among the most sought-after in the job market worldwide.

🧰 Massive Ecosystem

Hundreds of thousands of libraries on PyPI for every purpose imaginable.

🤖 AI & Data Science

NumPy, Pandas, TensorFlow, PyTorch — the AI world runs on Python.

🔄 Cross-platform

Write once, run on Windows, macOS, and Linux with no changes.

What Can You Build?

  • 🌐 Web Applications — Django, Flask, FastAPI
  • 📊 Data Analysis & Visualisation — Pandas, Matplotlib, Seaborn
  • 🤖 Machine Learning & AI — TensorFlow, PyTorch, scikit-learn
  • 🕷️ Web Scraping — BeautifulSoup, Scrapy
  • ⚙️ Automation & Scripting — automate repetitive tasks
  • 🎮 Games — Pygame
  • 📡 IoT & Microcontrollers — MicroPython on Raspberry Pi, ESP32
  • 🔒 Cybersecurity — scripting tools, penetration testing
ℹ️ Python vs Other Languages

Python is slower than compiled languages like C++ or Java when it comes to raw execution speed. But for most real-world applications — especially in data science, scripting, and web development — Python's speed is more than enough, and development time is far faster.

Installing Python & VS Code

Before we can write Python, we need two things: the Python interpreter and a good code editor. Let's set everything up from scratch.

Step 1: Install Python

Go to python.org/downloads and download the latest Python 3.x release for your operating system.

⚠️ Windows Users — Important!

During installation, check the box that says "Add Python to PATH" before clicking Install Now. This is essential — without it, your terminal won't find Python.

Verify the Installation

Open a terminal (Command Prompt on Windows, Terminal on macOS/Linux) and type:

Terminal
python --version # Should output something like: Python 3.12.0 python3 --version # On macOS/Linux, you may need to use python3

Step 2: Install VS Code

Visual Studio Code (VS Code) is a free, powerful code editor made by Microsoft. It's the most popular editor for Python development.

Download it from code.visualstudio.com and install it like any normal application.

Essential VS Code Extension

After opening VS Code, click the Extensions icon on the left sidebar (or press Ctrl+Shift+X). Search for and install:

  • Python by Microsoft — enables syntax highlighting, IntelliSense, debugging, and more
  • Pylance — enhanced language support (usually installed with Python extension)

Step 3: Configure VS Code for Python

  1. Open VS Code and press Ctrl+Shift+P (or Cmd+Shift+P on Mac)
  2. Type "Python: Select Interpreter" and press Enter
  3. Choose the Python version you just installed

The Integrated Terminal

VS Code has a built-in terminal. Open it with Ctrl+` (backtick). This is where you'll run your Python files.

Alternative: Python Online (No Install)

If you can't install Python right now, you can use these online platforms to follow along:

  • replit.com — full IDE in the browser
  • python.org/shell — interactive Python shell
  • Google Colab — Python notebooks (great for data science later)

Your First Program

Every programmer's journey starts with "Hello, World!" — a simple program that displays text on the screen. Let's write yours.

Creating a Python File

  1. Open VS Code and create a new folder called python-course
  2. Create a new file inside it called hello.py (the .py extension tells the computer it's Python)
  3. Type the following code:
Python
print("Hello, World!")

Run it in the terminal:

Terminal
python hello.py # Output: Hello, World!

Understanding print()

print() is a built-in function in Python. It displays whatever you put inside the parentheses on the screen. The text inside quotes is called a string.

Python
print("Hello, World!") # Prints text print(42) # Prints a number print(3.14) # Prints a decimal print("My name is", "Alex") # Prints two values separated by space print() # Prints a blank line

Comments in Python

Comments are notes for humans — Python ignores them completely. Use the # symbol to start a comment.

Python
# This is a comment — Python ignores this line print("This line runs") # Inline comment at end of code # Always comment your code! # It helps others (and future you) understand what the code does.

For multi-line comments, you can use triple quotes (though technically these are strings):

Python
""" This is a multi-line comment / docstring. It can span multiple lines. Often used to document functions and classes. """

The Interactive Python Shell (REPL)

Type python in your terminal to open the interactive shell (REPL = Read-Eval-Print Loop). You can type Python code and see results immediately — great for testing ideas.

Python REPL
$ python Python 3.12.0 ... >>> print("hello") hello >>> 2 + 3 5 >>> exit() # Type this to quit

output()
✅ Correct! print() is the built-in function for displaying output.
❌ Not quite. In Python, print() is the correct function for displaying output.

How Python Works

Understanding what happens under the hood when you run a Python program will make you a better programmer and help you debug errors more effectively.

Source Code → Bytecode → Execution

When you run a Python file, here's exactly what happens:

  1. You write source code in a .py file (human-readable text)
  2. The Python interpreter reads your file and checks for syntax errors
  3. Python compiles to bytecode — a lower-level, platform-independent format stored in .pyc files inside a __pycache__ folder
  4. The Python Virtual Machine (PVM) executes the bytecode line by line
ℹ️ Interpreted vs Compiled

Python is called "interpreted" because each line is processed at runtime rather than pre-compiled into machine code. This makes Python slower in raw speed but much faster in development. Languages like C or Rust compile ahead of time to machine code.

Indentation — Python's Special Rule

Python uses indentation (spaces/tabs) to define code blocks. This is unlike most languages that use curly braces {}. This makes Python code look clean, but you must be consistent.

Python
# CORRECT: 4 spaces for indentation (standard) if True: print("This is indented correctly") # WRONG: mixing spaces and tabs, or wrong number of spaces if True: print("2 spaces") print("4 spaces") # IndentationError!
💡 Best Practice

Always use 4 spaces per indentation level. VS Code does this automatically when you press Tab. Never mix tabs and spaces.

Python Syntax Rules

  • Python is case-sensitive: print ≠ Print ≠ PRINT
  • Statements end at the end of a line — no semicolons needed (though they work)
  • Long lines can be broken with a backslash \ or by wrapping in parentheses
  • Python executes from top to bottom, line by line
Python
# No semicolons needed print("Line 1") print("Line 2") # Long line continuation with backslash result = 1 + 2 + 3 + 4 \ + 5 + 6 # Or wrap in parentheses (preferred) result = (1 + 2 + 3 + 4 + 5 + 6)

Reading Error Messages

When Python encounters an error, it shows a traceback. Don't panic — learn to read it:

Error Example
Traceback (most recent call last): File "hello.py", line 3, in <module> prin("hello") NameError: name 'prin' is not defined

Reading this: On line 3, you called prin which doesn't exist — you probably meant print.

Common error types you'll see:

  • SyntaxError — invalid Python syntax (typo, missing colon, etc.)
  • NameError — using a variable or function that doesn't exist
  • TypeError — wrong data type for an operation
  • IndentationError — wrong indentation
  • ValueError — right type, wrong value
