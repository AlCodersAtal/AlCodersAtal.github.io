Defining Functions

Functions are named, reusable blocks of code. Write once, call anywhere — they keep your programs organised and maintainable.

Why Functions?

  • DRY — Don't Repeat Yourself. Write logic once, reuse everywhere.
  • Readable — calculate_tax() is clearer than 5 raw lines of math.
  • Testable — Small functions are easy to test in isolation.
  • Maintainable — Fix a bug in one place, not twenty.

def — Defining a Function

Python
# Define the function def greet(): """Prints a greeting message.""" print("Hello, World!") print("Welcome to Python.") # Call (run) the function greet() # Hello, World! / Welcome to Python. greet() # Call it again — code runs again
ℹ️ Docstrings

The triple-quoted string right after def is a docstring. It documents what the function does. Access it with help(greet). Always write docstrings for your functions!

Module 5 · Lesson 2

Parameters & Arguments

Parameters make functions flexible — pass in different data each time you call them.

Parameters vs Arguments

Python
# 'name' and 'age' are PARAMETERS (in the definition) def greet_person(name, age): print(f"Hello {name}! You are {age} years old.") # "Alice" and 25 are ARGUMENTS (passed at call time) greet_person("Alice", 25) greet_person("Bob", 30)

Positional vs Keyword Arguments

Python
def describe(animal, colour, size): print(f"A {colour} {size} {animal}") # Positional — order matters describe("cat", "black", "small") # Keyword — order doesn't matter describe(size="large", animal="dog", colour="brown")
Module 5 · Lesson 3

Return Values

The return statement sends a computed value back to whoever called the function.

Returning a Value

Python
def add(a, b): return a + b result = add(3, 4) print(result) # 7 print(add(1, 2) + add(3, 4)) # 10 — compose results

Returning Multiple Values

Python
def min_max(numbers): return min(numbers), max(numbers) # returns a tuple low, high = min_max([3, 1, 7, 9, 4]) print(f"Min: {low}, Max: {high}") # Min: 1, Max: 9

Early Return

Python
def divide(a, b): if b == 0: return None # exit early return a / b print(divide(10, 2)) # 5.0 print(divide(10, 0)) # None
Module 5 · Lesson 4

Default & Keyword Arguments

Default values make parameters optional, so callers only need to provide values when they want something different from the default.

Default Parameter Values

Python
def greet(name, greeting="Hello"): print(f"{greeting}, {name}!") greet("Alice") # Hello, Alice! greet("Bob", "Namaste") # Namaste, Bob!
⚠️ Never use mutable defaults (list, dict)

Use None as the default and create the mutable object inside the function. Mutable defaults are shared across all calls and cause subtle bugs.

Python
# WRONG def append_wrong(item, lst=[]): lst.append(item) return lst # CORRECT def append_right(item, lst=None): if lst is None: lst = [] lst.append(item) return lst
Module 5 · Lesson 5

*args & **kwargs

When you don't know how many arguments will be passed, *args and **kwargs let your function accept any number.

*args — Collect Extra Positional Args into a Tuple

Python
def add_all(*args): return sum(args) print(add_all(1, 2)) # 3 print(add_all(1, 2, 3, 4, 5)) # 15

**kwargs — Collect Extra Keyword Args into a Dict

Python
def print_info(**kwargs): for key, val in kwargs.items(): print(f" {key}: {val}") print_info(name="Alice", age=25, city="Kochi") # name: Alice # age: 25 # city: Kochi

Combining All Three

Python
def mega_func(pos1, *args, **kwargs): print(f"pos1: {pos1}") print(f"args: {args}") print(f"kwargs: {kwargs}") mega_func(1, 2, 3, x=10, y=20)
Module 5 · Lesson 6

Lambda Functions

Lambda functions are tiny anonymous functions defined in one line. Perfect as arguments to map(), filter(), sorted().

Lambda Syntax

Python
# Regular function def square(x): return x ** 2 # Equivalent lambda — lambda params: expression square = lambda x: x ** 2 print(square(5)) # 25 # Used inline with sorted() people = [("Alice", 25), ("Bob", 18), ("Charlie", 30)] sorted_people = sorted(people, key=lambda p: p[1]) print(sorted_people) # sorted by age # map() — apply to each element nums = [1, 2, 3, 4, 5] squares = list(map(lambda x: x**2, nums)) print(squares) # [1, 4, 9, 16, 25] # filter() — keep items where function returns True evens = list(filter(lambda x: x % 2 == 0, nums)) print(evens) # [2, 4]
Module 5 · Lesson 7

Scope & Closures

Scope determines where a variable can be seen. Closures are functions that remember the variables from their enclosing environment.

Local vs Global Scope

Python
x = 10 # GLOBAL variable def my_func(): y = 20 # LOCAL variable — only exists inside this function print(x) # Can read global x print(y) # Local y my_func() print(x) # 10 — global still accessible # print(y) # NameError — y doesn't exist here

global keyword

Python
count = 0 def increment(): global count # Tell Python we mean the global count count += 1 increment() increment() print(count) # 2

Closures

A closure is a function that captures variables from its outer scope, even after the outer function has returned.

Python
def make_multiplier(factor): def multiply(number): return number * factor # 'factor' from outer scope return multiply double = make_multiplier(2) triple = make_multiplier(3) print(double(5)) # 10 print(triple(5)) # 15
