Reading & Writing Files

Programs need to persist data beyond their runtime. Python makes file I/O simple and safe with the with statement.

Opening Files

Python
# WRITING — creates file if it doesn't exist with open("notes.txt", "w") as f: f.write("Hello, file!\n") f.write("Second line\n") # READING with open("notes.txt", "r") as f: content = f.read() # Read entire file as string print(content) # Read line by line (memory efficient for large files) with open("notes.txt") as f: for line in f: print(line.strip()) # Read all lines into list with open("notes.txt") as f: lines = f.readlines() # APPENDING — adds to end without overwriting with open("notes.txt", "a") as f: f.write("Appended line\n")

File Modes Summary

ModeMeaning
"r"Read (default) — error if file doesn't exist
"w"Write — creates file or overwrites existing
"a"Append — adds to end of file
"x"Create — error if file already exists
"rb"Read binary (images, PDFs)
"wb"Write binary

Working with CSV & JSON

CSV and JSON are the two most common data exchange formats. Python's standard library handles both beautifully.

CSV Files

Python
import csv # Writing CSV students = [ ["Name", "Age", "Score"], ["Alice", 20, 92], ["Bob", 21, 78], ] with open("students.csv", "w", newline="") as f: writer = csv.writer(f) writer.writerows(students) # Reading CSV with open("students.csv") as f: reader = csv.DictReader(f) # First row becomes keys for row in reader: print(f"{row['Name']} scored {row['Score']}")

JSON Files

Python
import json data = { "name": "Alice", "scores": [92, 87, 95], "active": True } # Write JSON to file with open("data.json", "w") as f: json.dump(data, f, indent=2) # indent=2 for pretty printing # Read JSON from file with open("data.json") as f: loaded = json.load(f) print(loaded["name"]) # Alice print(loaded["scores"]) # [92, 87, 95] # Convert to/from JSON string (without files) json_str = json.dumps(data) # dict → string back = json.loads(json_str) # string → dict

Try / Except / Finally

Exceptions are runtime errors. Instead of letting them crash your program, Python lets you catch and handle them gracefully.

Basic try/except

Python
# Without exception handling — crashes # x = int("hello") ← ValueError! # With exception handling try: x = int(input("Enter a number: ")) print(10 / x) except ValueError: print("That wasn't a number!") except ZeroDivisionError: print("Can't divide by zero!") except Exception as e: print(f"Something went wrong: {e}") else: print("All went well!") # Runs ONLY if no exception finally: print("This ALWAYS runs") # Cleanup code

Common Built-in Exceptions

ExceptionWhen it happens
ValueErrorWrong value: int("abc")
TypeErrorWrong type: "2" + 2
ZeroDivisionErrorDivision by zero
FileNotFoundErrorFile doesn't exist
KeyErrorDict key not found
IndexErrorList index out of range
AttributeErrorObject has no that attribute
ImportErrorModule not found

Custom Exceptions

Create your own exception classes to represent errors specific to your application. This makes your error messages meaningful and your code self-documenting.

Defining Custom Exceptions

Python
# Custom exceptions inherit from Exception class InsufficientFundsError(Exception): """Raised when bank account has insufficient funds.""" def __init__(self, balance, amount): self.balance = balance self.amount = amount super().__init__( f"Cannot withdraw £{amount}: balance is £{balance}" ) class BankAccount: def __init__(self, balance): self.balance = balance def withdraw(self, amount): if amount > self.balance: raise InsufficientFundsError(self.balance, amount) self.balance -= amount account = BankAccount(100) try: account.withdraw(200) except InsufficientFundsError as e: print(e) # Cannot withdraw £200: balance is £100

Context Managers

Context managers automate setup and teardown — opening/closing files, acquiring/releasing locks. The with statement uses them.

How with Works

When you write with open(...) as f:, Python automatically calls f.close() when the block exits — even if an exception occurs. This prevents resource leaks.

Python
# Without context manager — risky f = open("file.txt") data = f.read() f.close() # If an exception happens above, this never runs! # With context manager — safe and clean with open("file.txt") as f: data = f.read() # f.close() always called, exception or not

Building Your Own Context Manager

Python
from contextlib import contextmanager import time @contextmanager def timer(): """Context manager that measures execution time.""" start = time.time() yield # code in the 'with' block runs here elapsed = time.time() - start print(f"Elapsed: {elapsed:.3f}s") with timer(): total = sum(range(10_000_000))
