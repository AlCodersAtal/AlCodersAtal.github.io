Classes & Objects

Object-Oriented Programming (OOP) models real-world things as objects. A class is a blueprint; an object is an instance built from that blueprint.

Why OOP?

Imagine building a game with 50 enemy characters. Without OOP, you'd manage 50 × N variables. With OOP, you define an Enemy class once and create 50 objects from it — each with its own health, position, and name.

Class vs Object

Python
# CLASS — the blueprint class Dog: """A class representing a dog.""" species = "Canis lupus familiaris" # class attribute (shared) def __init__(self, name, breed, age): self.name = name # instance attributes (unique per object) self.breed = breed self.age = age def bark(self): print(f"{self.name} says: Woof!") def describe(self): print(f"{self.name} is a {self.age}-year-old {self.breed}") # OBJECTS — instances of the blueprint rex = Dog("Rex", "German Shepherd", 3) buddy = Dog("Buddy", "Golden Retriever", 5) rex.bark() # Rex says: Woof! buddy.describe() # Buddy is a 5-year-old Golden Retriever print(Dog.species) # Access class attribute

__init__ & self

__init__ is the constructor — it runs automatically when you create an object. self refers to the instance being created.

Understanding self

Python
class BankAccount: def __init__(self, owner, balance=0): # self.owner creates an instance attribute named 'owner' self.owner = owner self.balance = balance self.transactions = [] def deposit(self, amount): self.balance += amount self.transactions.append(f"+£{amount}") def withdraw(self, amount): if amount > self.balance: print("Insufficient funds!") return self.balance -= amount self.transactions.append(f"-£{amount}") def statement(self): print(f"Account: {self.owner} | Balance: £{self.balance}") print("Transactions:", self.transactions) acc = BankAccount("Alice", 100) acc.deposit(50) acc.withdraw(30) acc.statement()

Methods & Attributes

Classes have three kinds of methods — instance, class, and static — each with a different relationship to the class and its data.

Instance, Class, and Static Methods

Python
class Circle: pi = 3.14159 # class attribute def __init__(self, radius): self.radius = radius def area(self): # instance method — needs self return self.pi * self.radius ** 2 @classmethod def from_diameter(cls, diameter): # class method — alternative constructor return cls(diameter / 2) @staticmethod def is_valid_radius(r): # static method — utility, no self/cls return r > 0 c1 = Circle(5) print(c1.area()) # 78.53975 c2 = Circle.from_diameter(10) # radius=5 print(Circle.is_valid_radius(-1)) # False

Inheritance

Inheritance lets a class (child) acquire all attributes and methods of another class (parent), then extend or override them.

Single Inheritance

Python
class Animal: # Parent class def __init__(self, name, age): self.name = name self.age = age def speak(self): print("...") def describe(self): print(f"{self.name} is {self.age} years old") class Dog(Animal): # Dog inherits from Animal def __init__(self, name, age, breed): super().__init__(name, age) # call parent __init__ self.breed = breed def speak(self): # Override parent method print(f"{self.name}: Woof!") class Cat(Animal): def speak(self): print(f"{self.name}: Meow!") dog = Dog("Rex", 3, "Husky") cat = Cat("Whiskers", 5) dog.describe() # inherited from Animal dog.speak() # overridden — Rex: Woof! cat.speak() # Whiskers: Meow! print(isinstance(dog, Animal)) # True — dog IS an Animal
Module 9 · Lesson 5

Encapsulation & Polymorphism

Encapsulation hides internal details. Polymorphism lets different objects respond to the same method call in their own way.

Encapsulation — Private Attributes

Python
class Person: def __init__(self, name, age): self.name = name # public self._salary = 0 # protected (convention — don't touch outside) self.__age = age # private (name-mangled) @property def age(self): # getter — access like attribute return self.__age @age.setter def age(self, value): # setter — validate before setting if value < 0: raise ValueError("Age cannot be negative") self.__age = value p = Person("Alice", 25) print(p.age) # 25 — uses getter p.age = 30 # uses setter — validates

Polymorphism

Python
# Same method name, different behaviour animals = [Dog("Rex", 3), Cat("Milo", 2)] for animal in animals: animal.speak() # Each responds in its own way # Rex: Woof! # Milo: Meow!
Module 9 · Lesson 6

Dunder Methods

Dunder (double underscore) methods let your custom classes work with Python's built-in operators and functions like +, len(), str(), and more.

Common Dunder Methods

Python
class Vector: def __init__(self, x, y): self.x, self.y = x, y def __str__(self): # print(v) → "Vector(3, 4)" return f"Vector({self.x}, {self.y})" def __repr__(self): # repr(v) for debugging return f"Vector({self.x!r}, {self.y!r})" def __add__(self, other): # v1 + v2 return Vector(self.x + other.x, self.y + other.y) def __len__(self): # len(v) return int((self.x**2 + self.y**2) ** 0.5) def __eq__(self, other): # v1 == v2 return self.x == other.x and self.y == other.y v1 = Vector(1, 2) v2 = Vector(3, 4) print(v1 + v2) # Vector(4, 6) print(len(v2)) # 5
Module 9 · Lesson 7

Dataclasses

Python 3.7+ introduced @dataclass — a decorator that auto-generates __init__, __repr__, __eq__ and more for simple data-holding classes.

@dataclass in Action

Python
from dataclasses import dataclass, field @dataclass class Student: name: str age: int grade: str = "A" # default value scores: list = field(default_factory=list) # mutable default # __init__ is auto-generated — no need to write it! alice = Student("Alice", 20) bob = Student("Bob", 21, "B") print(alice) # Student(name='Alice', age=20, grade='A', scores=[]) print(alice == Student("Alice", 20)) # True — __eq__ auto-generated
