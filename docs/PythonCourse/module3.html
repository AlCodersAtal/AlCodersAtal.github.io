Augmented Assignment Operators

These combine an arithmetic operation with assignment — a shorthand for updating a variable.

Python
x = 10 x += 5 # x = x + 5 → 15 x -= 3 # x = x - 3 → 12 x *= 2 # x = x * 2 → 24 x //= 5 # x = x // 5 → 4 x **= 3 # x = x ** 3 → 64 print(x) # 64

Practical: Even or Odd?

The modulo operator is great for checking if a number is even or odd:

Python
num = int(input("Enter a number: ")) print(num % 2 == 0) # True if even, False if odd

Comparison Operators

Comparison operators compare two values and return a boolean (True or False). They're the backbone of all decision-making code.

All Comparison Operators

Python
a, b = 10, 5 print(a == b) # False — equal to print(a != b) # True — not equal to print(a > b) # True — greater than print(a < b) # False — less than print(a >= b) # True — greater than or equal print(a <= b) # False — less than or equal # Chained comparisons — unique to Python! age = 20 print(18 <= age < 65) # True — working age

== vs is

== checks if values are equal. is checks if they are the same object in memory. Use is only for None, True, False.

Python
x = [1, 2, 3] y = [1, 2, 3] print(x == y) # True — same values print(x is y) # False — different objects in memory print(None is None) # True — correct way to check None

Logical Operators

Logical operators combine multiple conditions. They let you ask questions like "is A true AND B true?" or "is A true OR B true?"

and, or, not

Python
# and — both must be True print(True and True) # True print(True and False) # False print(False and False) # False # or — at least one must be True print(True or False) # True print(False or False) # False # not — flips the boolean print(not True) # False print(not False) # True # Practical example age = 20 has_id = True print(age >= 18 and has_id) # True — allowed entry

Short-Circuit Evaluation

Python evaluates and/or lazily — it stops as soon as the result is known.

Python
# With 'and': if first is False, second is NOT evaluated x = 0 print(x != 0 and 10/x > 1) # False — no ZeroDivisionError! # With 'or': if first is True, second is NOT evaluated print(True or 1/0) # True — division never happens
Module 3 · Lesson 4

String Operations

Strings support several operator-like operations. Let's also introduce f-strings — the modern way to embed values in strings.

f-strings (Formatted String Literals)

f-strings (Python 3.6+) let you embed expressions directly inside strings. Prefix the string with f and put expressions in {}.

Python
name = "Alice" age = 25 score = 95.678 # Old way — messy print("Name: " + name + ", Age: " + str(age)) # f-string — clean and powerful print(f"Name: {name}, Age: {age}") print(f"Score: {score:.2f}") # 95.68 — 2 decimal places print(f"Double age: {age * 2}") # expressions work too print(f"{name.upper()} is {age} years old")

in and not in

Python
text = "Hello, World!" print("World" in text) # True print("Python" in text) # False print("Python" not in text) # True

Operator Precedence

When multiple operators appear in one expression, Python follows rules about which to evaluate first — just like BODMAS in maths.

Precedence Order (Highest to Lowest)

PriorityOperatorsDescription
1 (highest)()Parentheses
2**Exponentiation
3+x -xUnary plus/minus
4* / // %Multiplication, division, modulo
5+ -Addition, subtraction
6== != > < >= <=Comparisons
7notLogical NOT
8andLogical AND
9 (lowest)orLogical OR
Python
print(2 + 3 * 4) # 14 — * before + print((2 + 3) * 4) # 20 — () first print(2 ** 3 ** 2) # 512 — ** right-associative: 2**(3**2)=2**9=512 print(10 > 5 and 3 < 7) # True — comparisons before and

When in doubt, use parentheses to make your intent explicit. It's better to be clear than clever.
