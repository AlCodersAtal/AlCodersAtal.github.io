Importing Modules

A module is any .py file. Importing lets you use code from other files — whether from Python's standard library, third-party packages, or your own files.

import Styles

Python
# Import the whole module import math print(math.sqrt(16)) # 4.0 print(math.pi) # 3.14159... # Import specific names from math import sqrt, pi, floor print(sqrt(25)) # No need for math. prefix # Import with alias import numpy as np # Common convention import pandas as pd # Common convention from datetime import datetime as dt # Import everything — NOT recommended (pollutes namespace) from math import * # Avoid this in production code

__name__ == "__main__"

Python
# mymodule.py def add(a, b): return a + b # This block ONLY runs when this file is run directly # NOT when imported by another file if __name__ == "__main__": print(add(3, 4)) # test code here

Standard Library Tour

Python ships with a massive standard library — hundreds of modules covering almost every common task. You don't need to install anything extra.

Essential Standard Library Modules

Python
# math — mathematical functions import math print(math.sqrt(2), math.log(100), math.sin(math.pi/2)) # random — random numbers import random print(random.randint(1, 10)) # random integer 1-10 print(random.choice(["a","b","c"])) # random element random.shuffle(my_list) # shuffle in place # datetime — dates and times from datetime import datetime, timedelta now = datetime.now() print(now.strftime("%Y-%m-%d %H:%M")) tomorrow = now + timedelta(days=1) # os — operating system interface import os print(os.getcwd()) # current directory print(os.listdir(".")) # list files in directory os.makedirs("new_folder", exist_ok=True) # pathlib — modern path handling from pathlib import Path p = Path("notes.txt") print(p.exists()) print(p.suffix) # .txt p.read_text() # collections — useful data structures from collections import Counter, defaultdict, deque counts = Counter(["a", "b", "a", "c", "a"]) print(counts.most_common(2)) # [('a', 3), ('b', 1)]
Module 10 · Lesson 3

pip & Virtual Environments

pip is Python's package installer. Virtual environments keep project dependencies isolated so different projects don't conflict.

pip — Package Installer

Terminal
# Install a package pip install requests # Install specific version pip install requests==2.31.0 # Install from requirements file pip install -r requirements.txt # List installed packages pip list # Show info about a package pip show requests # Uninstall pip uninstall requests # Freeze installed packages to file pip freeze > requirements.txt

Virtual Environments — venv

Terminal
# Create a virtual environment python -m venv venv # Activate it # Windows: venv\Scripts\activate # macOS/Linux: source venv/bin/activate # You'll see (venv) in your prompt — now install freely! pip install flask pandas requests # Deactivate when done deactivate
💡 Always use virtual environments

Create a new virtual environment for every project. Add venv/ to your .gitignore. Share dependencies via requirements.txt. This prevents version conflicts between projects.

Module 10 · Lesson 4

Building Your Own Module

Any .py file is a module. Organise your code into modules to keep projects maintainable and shareable.

Creating a Module

mathutils.py
""" mathutils.py — A collection of useful math utilities. """ def is_prime(n): """Return True if n is a prime number.""" if n < 2: return False for i in range(2, int(n**0.5) + 1): if n % i == 0: return False return True def factorial(n): """Return n! (factorial).""" if n <= 1: return 1 return n * factorial(n - 1) if __name__ == "__main__": print(is_prime(17)) # True print(factorial(5)) # 120
main.py — importing your module
from mathutils import is_prime, factorial for n in range(2, 20): if is_prime(n): print(n, end=" ") # 2 3 5 7 11 13 17 19
